### 1.3 Phase 4 关键教训

1. **VPD 是唯一有效的新特征组**（+4.6% R²），PPF 几乎无独立贡献
2. **简单模型 > 复杂模型**（304 样本）：GBR > ConstrainedGBR > HRL > GNN
3. **Stacking 在 304 样本上 3 次失败**（Phase 1/2/3），但 7,486 样本可能改变局面
4. **GNN 端到端完全不可行**（参数/样本比 > 300:1），嵌入→GBR 融合是可行折中
5. **Nested CV 是唯一可信的评估方法**，简单 train/test split 会高估性能

## 二、算法候选者分析

### 2.1 树模型家族（主力候选）

| 模型 | 7.5K 样本适用性 | 调研推荐度 | 核心优势 | 核心劣势 |
|------|----------------|-----------|---------|---------|
| **CatBoost** | 最优区间 | ⭐⭐⭐⭐⭐ | Ordered boosting 防过拟合；默认参数即强；GPU 加速 | 训练稍慢于 LightGBM |
| **LightGBM** | 优 | ⭐⭐⭐⭐ | 极快训练；低内存；GPU 加速成熟 | 叶子优先策略在中等数据上不如 CatBoost |
| **XGBoost** | 优 | ⭐⭐⭐⭐ | 最成熟生态；GPU 支持好 | 2024+ 基准中已非最优 |
| **ExtraTrees** | 中 | ⭐⭐⭐ | 极快；天然正则化 | 大数据集不如 boosting |
| **GBR (sklearn)** | 差 | ⭐⭐ | 简单可靠 | 无 GPU；被 CatBoost 全面超越 |

**关键证据**: CatBoost 在 2024-2025 多项独立基准中一致领先中等数据集。Ordered boosting 在 1K-50K 样本上有效缓解 target leakage。

### 2.2 深度学习模型

| 模型 | 适用性 | 推荐度 | 角色 |
|------|--------|--------|------|
| **TabPFN v2** | ⭐⭐⭐⭐⭐ | 5/5 | <10K 样本首次系统性超越 GBDT 的 DL；零调参 |
| **TabM** (ICLR 2025) | ⭐⭐⭐⭐ | 4/5 | 参数高效 MLP 集成，与 GBDT 持平 |
| GNN (端到端) | ⭐⭐ | 2/5 | Phase 4 已证明 7.5K 仍不够（E9-E15 全败） |
| GNN (嵌入→GBR) | ⭐⭐⭐ | 3/5 | 可作为 stacking 基学习器的特征来源 |

**TabPFN v2 亮点**:
- 限制：<10,000 样本，<500 特征维度 → 我们的 7,500 + 46d 完美匹配
- 零调参 → Nested CV 内层可跳过超参搜索，节省 90% 计算时间
- 与树模型视角互补（attention vs decision boundary）

### 2.3 AutoML 框架

| 框架 | 2024 Kaggle 表现 | 适用性 | 角色 |
|------|-----------------|--------|------|
| **AutoGluon** | 15/18 奖牌, 7 金 | ⭐⭐⭐⭐⭐ | 多层 stacking + k-fold bagging → 性能上界参考 |

**AutoGluon 的价值**：快速建立性能上界（"人工方案能接近 AutoGluon 就说明没有遗漏"）。

### 2.4 不确定性量化 (UQ)

| 方法 | 推荐度 | 说明 |
|------|--------|------|
| **MAPIE CQR** | ⭐⭐⭐⭐⭐ | 分布无关；自适应区间宽度；sklearn 兼容；论文亮点 |
| NGBoost | ⭐⭐⭐ | 概率梯度提升，但假设正态性 |
| Deep Ensemble | ⭐⭐⭐ | Phase 4 E14 已验证，R²=-0.34（端到端失败，但嵌入融合版可考虑）|

**核酸域外预测必须有 UQ**：CQR 区间会在域外样本上自然变宽，这正是论文需要的。

### 方案 B：Tandem-M2M GNN 嵌入融合（基于 Phase 5 结果修订）

> **修订日期**: 2026-03-15
> **修订依据**: Phase 5 E16-E24 实验结果（--no-optuna 版）
> **与方案 A 关系**: 完全独立，方案 A（E16-E24）已完成，方案 B 实验编号从 E25 起

**描述**: 在方案 A 已验证的表格管线基础上，引入物理嵌入 GNN（Tandem-M2M）提取 64 维图结构嵌入，与 M2M-V 46 维拼接为 110 维表示，输入方案 A 已验证有效的 Stacking 框架。核心创新：GNN 不直接预测 Tg，而是学习 Van Krevelen GC 基线的残差（物理残差学习）。

#### B.1 Phase 5 结果对方案 B 的关键启示

| Phase 5 发现 | 对方案 B 的启示 |
|-------------|---------------|
| TabPFN v2 R²=0.8955 领先所有树模型 | GNN 嵌入的目标不是取代 TabPFN，而是提供互补的**结构表示**维度 |
| Stacking v2 R²=0.8750 有效（7.5K 样本） | GNN 嵌入可安全接入 Stacking 框架，不必担心过拟合（Phase 4 304 样本的核心瓶颈已消除） |
| VPD 贡献 6/15 Top 特征（RDKit 三聚体描述符） | GNN 嵌入应超越 RDKit 描述符，捕捉**非线性结构-性质关系**（VPD-Deep 概念） |
| FlexibilityIndex SHAP=41.51 统治性 | GNN 原子特征需注入柔性键标注，确保 GNN 不会丢失最重要的物理信息 |
| Top-15 = 全量 46 维性能 → 31 维冗余 | GNN 64 维嵌入必须编码**不同于**现有 46 维的信息才有增量价值 |
| CrossConformal CatBoost 是最佳 UQ | 方案 B 可复用此 UQ 方法，也可升级为 Deep Ensemble + Conformal |

**Phase 4 E12 失败的根因分析（不再适用）**:
- Phase 4 E12 (R²=0.839 < E3 R²=0.866) 的失败原因：304 样本 GNN 预训练数据不足，嵌入噪声大于信号
- Phase 5 条件变化：7,486 主集 + 13,400 外部 + 46,000 Fox = **~67K 预训练数据**，比 Phase 4 增 220 倍
- 结论：Phase 4 的负面结果不能推广到 Phase 5

#### B.2 架构设计（Tandem-M2M + Stacking 融合）

```
分支 1 (表格 — 方案 A 已验证):
  SMILES → feature_pipeline(M2M-V) → 46d → PowerTransformer + MinMaxScaler

分支 2 (图 — 本方案新增):
  SMILES → build_oligomer(n=3) → 分子图 G(V,E)
       ↓
  物理增强原子特征 (25d):
    标准 15d (element, charge, degree, aromatic, ring)
    + 柔性键标注 + GC 基团类型 + 主链/侧链标注 + 局部立体
       ↓
  Tandem-M2M:
    ├─ 物理分支: Van Krevelen GC → Tg_GC (确定性基线，不训练)
    └─ 学习分支: 3-layer GAT (25→128→128→64) + GRIN 池化 → 64d 嵌入
       ↓
  GNN 输出: Tg = Tg_GC + alpha * GNN_residual  (alpha 可学习)
  嵌入提取: 64d 图级表示 (GRIN 重复不变池化)
       ↓
两分支拼接: 46d + 64d = 110d
       ↓
L0 基学习器 (方案 A Stacking v2 框架 + GNN 嵌入):
  ├─ CatBoost (110d, Optuna 调参)
  ├─ TabPFN v2 (110d, 零调参)        ← Phase 5 最强单模型，必须保留
  ├─ LightGBM (110d, Optuna 调参)
  └─ ExtraTrees (110d)
       ↓
L1 元学习器: Ridge
       ↓
UQ: MAPIE CrossConformal (方案 A 已验证) 或 Deep Ensemble ×5
```

#### B.3 关键技术创新点

**1. 物理残差学习（核心创新）**
GNN 不从头学 Tg，而是学 Van Krevelen GC 基线的残差。优势：
- 减小学习目标方差（残差 << Tg 本身），小数据更稳定
- 物理基线保证预测在合理范围内，尤其对核酸等域外样本
- alpha 参数可解释：物理 vs 学习的贡献比例

**2. GRIN 重复不变池化**
标准 GNN 的根本问题：3-RU 和 60-RU 输出不同嵌入（混淆"链更长"和"化学不同"）。
GRIN 策略：只对中间 RU 的原子做 max-pool，结果不随链长变化。

**3. VPD-Deep（GNN 版虚拟聚合描述符）**
```
原版 VPD:  RDKit(trimer)/3 - RDKit(monomer) → 12d (线性差值)
VPD-Deep:  GNN(trimer)/3 - GNN(monomer) → 64d (非线性差值)
```
GNN 版 Δ 嵌入能捕捉聚合过程的非线性结构变化，比 RDKit 描述符更丰富。

**4. 物理增强原子特征**
在标准化学特征之上注入物理知识，弥补纯数据驱动 GNN 的物理盲区：
- `is_flexible_bond_atom`: FlexibilityIndex 是 SHAP 第一特征，GNN 不能丢失
- `gc_group_type`: Van Krevelen 基团类型（供 GC 物理分支使用）
- `is_backbone_atom` / `is_sidechain_atom`: 主侧链区分（Tg 主要由主链决定）

#### B.4 预训练策略（质量分层）

```
Stage 1 — 大规模预训练 (A800 GPU):
  Layer 1 (高质量): PolyMetriX 7,367 + NeurIPS 过滤后 ~14,000 → ~13,400 实验 Tg
  Layer 2 (增强):   Fox 虚拟共聚物 ~46,000 (Fox 方程计算 Tg)
  Layer 3 (可选):   Kuenneth 共聚物 ~18,445
  ─── 合计: ~59,400-77,800 条 ───
  任务: Tg 预测 + GC 参数预测 (辅助任务)
  参数: epochs=100, lr=1e-3, batch=256, dropout=0.2

Stage 2 — 微调 (在统一集 7,486 条上):
  冻结 GAT 前两层，只训练 conv3 + MLP + alpha
  参数: epochs=50, lr=1e-4, batch=32, dropout=0.3, early_stop patience=10

Stage 3 — 嵌入提取:
  训练好的 GNN → 提取 64d 嵌入 → 与 M2M-V 46d 拼接 → 输入 Stacking

防泄漏: 预训练数据排除统一集 7,486 条的规范化 SMILES
```

### 方案 C：AutoGluon 全自动 + MAPIE CQR

**描述**: 直接用 AutoGluon 的多层 stacking 策略，在其输出上套 MAPIE CQR。

**架构**:
```
特征: M2M-V 46d
     ↓
AutoGluon(preset="best_quality"):
  L0: [CatBoost, LightGBM, XGBoost, ExtraTrees, NN, KNN] × 8-fold bagging
  L1: 加权集成
  L2: 最终 blending
     ↓
UQ: MAPIE CQR (AutoGluon 预测 → 校准)
```

**优势**:
- 2024 Kaggle 15/18 奖牌的策略，性能上界最高
- 几乎零代码（~20 行），AutoGluon 自动处理预处理/特征工程/模型选择
- 内置 k-fold bagging 防过拟合

**劣势**:
- 黑盒，难以在论文中解释具体贡献
- 论文价值低（"我用了 AutoML 就好了"不是好的研究贡献）
- 不可控：AutoGluon 的模型组合难以精确分析
- 无法展示 VPD 等特征工程的独立贡献
- 训练时间长（best_quality 需要几小时）

**预期**: R² ≈ 0.91-0.95

**复杂度**: 最低（实现），最高（分析）

---

### 方案 D：TabPFN v2 主力 + 轻量 Ensemble

**描述**: 以 TabPFN v2 为核心（零调参），辅以 CatBoost 的简单加权平均。

**架构**:
```
特征: M2M-V 46d
     ↓
并行预测:
  ├─ TabPFN v2 (零调参) → pred_1
  └─ CatBoost (Optuna 调参) → pred_2
     ↓
加权平均: α * pred_1 + (1-α) * pred_2 (α 通过 CV 选择)
     ↓
UQ: MAPIE CQR
```

**优势**:
- 最简洁的架构，实验可快速完成
- TabPFN v2 在 <10K 上首次超越 GBDT（论文亮点）
- 无 Stacking 过拟合风险
- 训练极快（TabPFN v2 推理式，CatBoost 也快）

**劣势**:
- TabPFN v2 是 2024 新模型，成熟度不如树模型
- 加权平均的提升幅度可能有限
- 缺少多样化基学习器的互补效应

**预期**: R² ≈ 0.89-0.92

**复杂度**: 最低

## 七、评估策略

### 7.1 主评估：Nested CV on Training Set

```
外层: RepeatedKFold(n_splits=5, n_repeats=3) = 15 folds
内层: Optuna 50 trials / TabPFN 跳过 / 或 KFold(3) + RandomizedSearch
指标: R², MAE, RMSE (mean ± std across 15 folds)
```

**为什么在训练集上做 Nested CV 而非全部 7,486 条**:
- 保留 1,498 条 holdout test 作为最终报告指标
- 论文需要 "unseen data" 的性能数字
- Nested CV 评估模型选择的泛化性能，holdout 评估最终模型

### 7.2 辅助评估：Holdout Test Set

```
训练: 5,988 条 (train split) → 最优模型 + 最优超参
测试: 1,498 条 (test split) → 单次评估
指标: R², MAE, RMSE (单点估计)
```

### 7.3 核酸迁移评估

```
桥梁聚合物 205 条 → 加权训练 → 核酸 Tg 预测
UQ: MAPIE CQR 区间宽度 → 核酸预测的可信度
```

## 八、特征策略

### 8.1 基准特征层：M2M-V 46d

Phase 4 已验证 M2M-V 是最优特征层（VPD +4.6% R²，PPF 无增益）。

Phase 5 保持 M2M-V 46d 作为基准：
- L0_Afsordeh: 4d（FlexibilityIndex, MolWt, RingCount, AromaticRings）
- L1_RDKit_2d: 15d（TPSA, MolLogP, HBD, HBA, RotBonds, ...）
- HBond: 15d（H-bond 供体/受体分布）
- VPD: 12d（虚拟聚合描述符 — 三聚体特征）
- 总计: 46 维

### 8.2 可选特征增强（消融实验 E23 探索）

- **Boruta-SHAP 特征精选**: 从 46d 中选 Top-K 最重要特征
- **M2M 56d 对比**: 加回 PPF 10d，看大数据集上 PPF 是否有增益（304 条没有）
- **Morgan 指纹 1024d**: 高维指纹在大数据集上可能有效（304 条时过拟合风险）
