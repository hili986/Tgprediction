## 零、行动哲学

### "有效数据极致精度" 原则

**不追求 100% 覆盖率，追求有效数据点上的最好结果。**

| 传统做法 | 本方案做法 |
|---------|-----------|
| 每个分子必须有完整特征向量 | 缺失值就是信息——NaN 告诉模型"这个物理量对这个分子不可靠" |
| 计算失败 → 填默认值 → 掩盖问题 | 计算失败 → 填 NaN → 模型用其他特征补偿 |
| 5-mer 构建失败 30% → 放弃这个特征 | 5-mer 对 70% 分子有效 → 这 70% 的预测精度大幅提升，其他 30% 靠已有特征 |
| GC 覆盖率 75% → 不敢用残差学习 | 对这 75% 做残差学习精度更高，其他 25% 用标准目标 |
| 每步 R² 没提升 → 立即回退 | 全量推进 → 最终集成时才判断贡献 → 单独无效的特征在组合中可能有效 |

### 技术支撑

- **CatBoost 原生支持 NaN**：自动学习缺失值的最优处理方式
- **TabPFN 支持 NaN**：meta-learner 见过各种缺失模式
- **oligomer_level 元特征**：告诉模型这条数据的链段信息质量 (0=计算失败, 3=3-mer, 5=5-mer)
- **gc_coverage 元特征**：告诉模型 GC 预测值是否可信

## 一、多尺度信息金字塔与因果链

### Tg 的物理因果链（三大理论统一视角）

```
原子结构 (键类型、杂化、电负性)
    ↓ 决定
旋转势垒 (每个可旋转键的 gauche↔trans 势垒)
    ↓ 决定（Gibbs-DiMarzio: f = 可访问构象数）
构象熵 S_c (链的构象自由度)
    ↓ 与
分子间力 E_h (CED: 范德华 + 偶极 + 氢键)
    ↓ 共同决定（Adam-Gibbs: τ = τ₀ × exp(C/T·S_c)）
自由体积需求 (协同运动需要的空间)
    ↓ 对应（自由体积理论: f → f_g ≈ 0.025 时冻结）
临界温度 Tg
```

## 二、逐特征详细设计

### 2.1 修正 M/f 柔性键计数 — 精细化 FlexibilityIndex

**物理依据**: Schneider-DiMarzio M/f 模型: Tg = A × (M/f) + C。预测精度 MAE 3.7-6.4K (100+ 聚合物验证)。

**现有实现的问题** (`physical_proxy.py` 第 56-92 行):

| 键类型 | 当前权重 | 正确权重 | 来源 |
|--------|---------|---------|------|
| C-C / C-O / C-N / C-S 单键 | 1.0 | 1.0 ✅ | Schneider 2006 |
| Si-O | 1.5 | 1.5 ✅ | Schneider 2006 |
| 酰胺 C-N | 0.0 | 0.0 ✅ | 共振半双键 |
| 芳-芳单键 | 0.5 | 0.5 ✅ | 受限旋转 |
| **C-OH 键** | **1.0 ❌** | **0.0** | 旋转不改变分子形状 |
| **C-CH₃ 键** | **1.0 ❌** | **0.0** | 甲基旋转不改变形状 |
| **对位苯环穿链** | **未实现 ❌** | **1.5** | 围绕苯环轴旋转柔性高 |
| **防零保护** | max(f, 0.5) | max(f, 0.1) | 0.5 过高，掩盖了真实刚性差异 |

**注**: 对位苯环穿链 (p-phenyl in backbone, f=1.5) 需要检测苯环的两个连接点是否在主链路径上且为 para 位。实现依赖 `_find_backbone_atoms()` 的准确性，在 Phase B 实现时如果骨架检测不够稳健，先跳过此规则，在后续迭代中补上。

**具体修改** (改 `_count_flexible_bonds`):

```python
def _count_flexible_bonds_v2(mol):
    """修正后的柔性键计数 (Schneider-DiMarzio 2006 规则)"""
    f_total = 0.0
    for bond in mol.GetBonds():
        if bond.IsInRing():
            continue
        if bond.GetBondType() != Chem.BondType.SINGLE:
            continue

        a1, a2 = bond.GetBeginAtom(), bond.GetEndAtom()
        sym1, sym2 = a1.GetSymbol(), a2.GetSymbol()

        # 规则 1: 酰胺 C-N → 0 (已有)
        if _is_amide_adjacent(mol, bond.GetIdx()):
            continue

        # 规则 2: C-OH, C-CH3 → 0 (新增)
        # 末端原子 (degree=1) 的旋转不改变分子形状
        if a1.GetDegree() == 1 or a2.GetDegree() == 1:
            continue

        # 规则 3: Si-O → 1.5 (已有)
        if (sym1 == 'Si' and sym2 == 'O') or (sym1 == 'O' and sym2 == 'Si'):
            f_total += 1.5
        # 规则 4: 芳-芳 → 0.5 (已有)
        elif a1.GetIsAromatic() and a2.GetIsAromatic():
            f_total += 0.5
        # 规则 5: 标准可旋转键 → 1.0
        else:
            f_total += 1.0

    return max(f_total, 0.1)  # 防零保护降低阈值
```

**新增特征**:
```
RBP_Mf_corrected = MolWt / _count_flexible_bonds_v2(mol)   ← 修正后的 M/f
RBP_flex_density = _count_flexible_bonds_v2(mol) / HeavyAtomCount  ← 修正后的柔性密度
```

**验证方法**:
- 对 Bicerano 数据集 304 条计算修正前后 M_per_f
- 计算与实验 Tg 的 Pearson 相关系数：修正后应 > 修正前
- 预期: r(M/f, Tg) 从 ~0.6 提升到 ~0.7

**计算成本**: 零额外成本 (修改现有函数)
**失败条件**: 若相关性无提升 → 说明被修正的键类型在数据集中太少

### 2.2 Boltzmann 加权有效构象数 N_eff — 编码 Gibbs-DiMarzio 核心变量

**物理依据**: Gibbs-DiMarzio 理论的 f 参数（每个键的可用构象数）是 Tg 的核心决定因素。FlexibilityIndex 用 2D 拓扑近似 f，我们要用 3D 构象采样直接测量。

**实现方案** (已验证可行):

```python
def compute_boltzmann_neff(smiles, n_confs=50, temperatures=(300.0, 500.0)):
    """
    Boltzmann 加权的有效构象数。

    物理意义:
      N_eff = exp(S_conf/kB) = exp(-Σ pᵢ ln pᵢ)
      等于 Shannon 信息论中的 "困惑度" (perplexity)

      N_eff 大 → 构象多样 → 链柔 → Tg 低  (Gibbs-DiMarzio)
      N_eff ≈ 1 → 只有一个优势构象 → 链刚 → Tg 高

    关键设计决策:
      - 用 3-mer 寡聚体而非单体 (捕捉邻基效应)
      - 双温度 (300K, 500K) 分别采样
      - 比值 N_eff(500K)/N_eff(300K) 反映能量景观陡峭度

    力场选择:
      MMFF94: 47% 构象在 1 kcal/mol 内 (Halgren 基准)
      已知弱点: 芳基酰胺势垒过高
      对我们够用: 需要的是 N_eff 的相对排序而非绝对值
    """
    # 对 3-mer 做构象采样 (比单体更物理)
    oligomer = build_oligomer(smiles, n=3)
    if oligomer is None:
        # 3-mer 失败 → NaN, 不回退到单体
        # 单体的 N_eff 缺少邻基效应, 物理意义不同, 不混用
        return {f'Neff_{int(T)}K': np.nan for T in temperatures} | {
            'Neff_ratio': np.nan, 'conf_strain': np.nan, 'neff_quality': 0
        }

    mol = Chem.MolFromSmiles(oligomer)
    if mol is None:
        return {f'Neff_{int(T)}K': np.nan for T in temperatures} | {
            'Neff_ratio': np.nan, 'conf_strain': np.nan, 'neff_quality': 0
        }
    mol = Chem.AddHs(mol)

    # 关键参数: useRandomCoords=True 使失败率接近零
    params = AllChem.ETKDGv3()
    params.randomSeed = 42
    params.useRandomCoords = True
    cids = AllChem.EmbedMultipleConfs(mol, numConfs=n_confs, params=params)

    if len(cids) < 3:
        return {f'Neff_{int(T)}K': np.nan for T in temperatures} | {
            'Neff_ratio': np.nan, 'conf_strain': np.nan,
            'neff_quality': 0  # 元特征: 计算失败
        }

    # MMFF 优化 + 收集能量
    energies = []
    for cid in cids:
        try:
            res = AllChem.MMFFOptimizeMolecule(mol, confId=cid, maxIters=500)
            props = AllChem.MMFFGetMoleculeProperties(mol)
            if props:
                ff = AllChem.MMFFGetMoleculeForceField(mol, props, confId=cid)
                if ff:
                    energies.append(ff.CalcEnergy())  # kcal/mol
        except Exception:
            continue

    if len(energies) < 3:
        return {f'Neff_{int(T)}K': np.nan for T in temperatures} | {
            'Neff_ratio': np.nan, 'conf_strain': np.nan, 'neff_quality': 0
        }

    energies = np.array(energies)
    E_min = energies.min()
    dE_kcal = energies - E_min

    results = {}
    for T in temperatures:
        RT_kcal = 1.987e-3 * T  # kcal/(mol·K) × K = kcal/mol
        boltz = np.exp(-dE_kcal / RT_kcal)
        Z = boltz.sum()
        p = boltz / Z
        S = -np.sum(p * np.log(p + 1e-30))
        N_eff = np.exp(S)
        results[f'Neff_{int(T)}K'] = N_eff

    # 比值特征: 反映能量景观的 "陡峭度"
    # 比值大 → 高温才解锁新构象 → 能量景观陡 → fragile 型 → Tg 可能异常
    results['Neff_ratio'] = results['Neff_500K'] / max(results['Neff_300K'], 1.0)

    # 最低能与平均能的差 → 构象应变能
    results['conf_strain'] = np.mean(energies) - E_min  # kcal/mol
    results['neff_quality'] = 1  # 元特征: 计算成功

    return results
```

**特征维度**: 4 (Neff_300K, Neff_500K, Neff_ratio, conf_strain)

**物理预测 (可验证)**:
- Neff_300K 与 Tg 应强负相关 (r < -0.5): 构象多 → 柔 → Tg 低
- Neff_ratio > 2 的分子应对应 fragile 型玻璃形成体
- conf_strain 与 Tg 应正相关: 高应变 → 构象受限 → Tg 高

**计算成本**: 50 构象/3-mer × 7486 分子, MMFF94 CPU 16 核并行
  - 瓶颈: ETKDGv3 构象生成 (CPU), 大分子 3-mer 可达 80s/mol
  - 能量计算: MMFF94 (CPU, RDKit 内置), 排序精度足够
  - 预计总耗时: 8-10 小时 (服务器后台 nohup 运行)
  - checkpoint/resume: 每 100 条保存, 中断后重启自动续传

**实际实现**: N_eff 和 C_n 合并为单函数 `compute_3mer_physics()`
  一次 3-mer 构建 + 构象采样同时产出 8 维特征, 避免重复计算

**能量后端**: MMFF94 (CPU) 默认, AIMNet2 (GPU) 可选 opt-in
  - 默认 MMFF94: RDKit 内置, 纯 CPU 16 核无冲突, 稳定可靠
  - 瓶颈是 ETKDGv3 构象生成 (CPU), 不是能量计算, 换 GPU 后端收益有限
  - MMFF94 的能量排序对 Boltzmann N_eff 足够 (需要相对排序, 非绝对精度)
  - AIMNet2 opt-in: 设置 `_BACKEND = None` 触发自动检测

**已知局限**: MMFF94 对芳基酰胺势垒过高 (Guba 2019)。
  作为排序特征仍可用, 模型从其他特征补偿。

**验证方案**:
1. 先对 20 个已知 Tg 的简单聚合物 (PE, PS, PMMA, PC, PDMS...) 计算 N_eff
2. 检查 N_eff 排序是否与 Tg 排序反相关
3. 如果 Spearman ρ(N_eff, Tg) > -0.4 → 特征有效
4. 如果 ρ > -0.2 → MMFF 排序精度不够, 考虑 opt-in AIMNet2 或 xTB

### 2.3 Van Krevelen GC Tg 预测值 — 物理基线

**物理依据**: GC 法 Tg = ΣYgi/Mw, Liu et al. 2020 达到 R²=0.9925。这是一个**极强的物理基线**。

**实现挑战与解决方案**:

| 挑战 | 方案 |
|------|------|
| 完整 58 基团 SMARTS 不存在 | 从现有 13 基团 (`solubility_param.py`) 起步，逐步扩展到 ~25 基团 |
| Yg 值需从论文 SI 提取 | 从 Cao 2020 PMC 全文和 VK 计算器交叉验证 |
| 覆盖率可能 < 100% | 对无法分解的分子标记 `gc_coverage=0`，作为元特征 |

**简化版 GC 实现** (利用已有 `solubility_param.py` 框架):

```python
# ⚠️ 以下 Yg 值是占位估算, 不可直接用于最终模型!
# 交叉验证显示 PMMA/PVC/PIB 偏差 > 100K
# Phase B 第一步: 从 Cao 2020 SI Table S3 或 VK 在线计算器提取真实 Yg 值
# 在真实 Yg 值验证通过之前, GC_Tg 特征标记为 experimental
TG_GC_GROUPS = {
    # SMARTS:           (name,       Ecoh,   Vw,    Yg)
    "[CH3]":            ("CH3",      4710,   33.5,  2.7),    # -CH₃
    "[CH2]":            ("CH2",      4940,   16.1,  2.7),    # -CH₂-
    "[CH1]":            ("CH",       3430,   -1.0,  -1.5),   # >CH-
    "[CX4H0]":          ("C_quat",   1470,   -19.2, -8.5),   # >C<
    "c1ccccc1":         ("phenyl",   31940,  52.4,  35.0),   # -C₆H₄- (对位)
    "[CX3](=O)[OX2]":  ("ester",    18000,  18.0,  17.0),   # -COO-
    "[CX3](=O)":       ("carbonyl", 17370,  10.8,  12.0),   # >C=O
    "[OX2H0]":         ("ether",    3350,   3.8,   4.0),    # -O-
    "[OX2H]":          ("hydroxyl", 21850,  10.0,  14.0),   # -OH
    "[NX3]([CX3]=O)":  ("amide",    33490,  9.5,   42.0),   # -NHCO-
    "[F]":             ("F",        2280,   18.0,  -0.5),   # -F
    "[Cl]":            ("Cl",       None,   None,  12.0),   # -Cl
    "[SiH0]([OX2])":   ("siloxane", None,   None,  -5.0),   # -Si(R)-O-
    # ... 扩展到 ~25 个常见基团
}

def compute_gc_tg(smiles):
    """计算 Van Krevelen 基团贡献法 Tg 预测值"""
    mol = Chem.MolFromSmiles(smiles.replace("*", "[H]"))
    if mol is None:
        return None, 0.0

    total_yg = 0.0
    matched_atoms = set()
    n_groups_matched = 0

    # 按基团优先级匹配: 大基团优先, 避免双重计数
    # 例如 苯环(6原子) 优先于 C-C(2原子), 酯基优先于羰基
    # 排序: SMARTS 匹配原子数多的优先
    sorted_groups = sorted(TG_GC_GROUPS.items(),
                           key=lambda x: len(Chem.MolFromSmarts(x[0]).GetAtoms()),
                           reverse=True)

    for smarts, (name, ecoh, vw, yg) in sorted_groups:
        pattern = Chem.MolFromSmarts(smarts)
        matches = mol.GetSubstructMatches(pattern)
        for match in matches:
            # 只计数尚未被更大基团覆盖的匹配
            if not any(a in matched_atoms for a in match):
                total_yg += yg
                matched_atoms.update(match)
                n_groups_matched += 1

    mw = Descriptors.MolWt(mol)
    coverage = len(matched_atoms) / max(mol.GetNumHeavyAtoms(), 1)

    if coverage < 0.3 or mw < 10:
        return None, coverage

    tg_gc = 1000 * total_yg / mw  # K
    return tg_gc, coverage
```

**输出特征** (2 维):
```
GC_Tg:       Van Krevelen 预测值 (K)。覆盖率<30% 时为 NaN, 模型自动处理。
GC_coverage: 基团覆盖率 (0-1)。告诉模型"这条 GC 预测可信吗"。
```

**Yg 值交叉验证** (实现前必须通过):

| 聚合物 | 重复单元 | 基团分解 | ΣYg | Mw | Tg_GC (K) | Tg_exp (K) | 误差 |
|--------|---------|---------|-----|-----|-----------|------------|------|
| PE | -CH₂CH₂- | 2×CH2 | 5.4 | 28 | 193 | 195 | -2K ✓ |
| PS | -CH₂CH(C₆H₅)- | CH2+CH+phenyl | 36.2 | 104 | 348 | 373 | -25K ⚠️ |
| PMMA | -CH₂C(CH₃)(COOCH₃)- | CH2+C_quat+CH3+ester | 12.9 | 100 | 129 | 378 | ❌ |
| PVC | -CH₂CHCl- | CH2+CH+Cl | 13.2 | 62.5 | 211 | 354 | ❌ |

**问题**: PS 偏差 25K 可接受, 但 PMMA 和 PVC 偏差巨大 → 当前 Yg 表缺少关键基团:
- PMMA 需要: -C(CH₃)- (α-甲基 on backbone) 和 -COOCH₃ 的正确分解
- PVC 需要: -CHCl- 的 Yg (极性取代基效应)

**解决**: 必须从 Cao 2020 SI (Table S3) 提取完整 Yg 表, 或从 VK 计算器反推。
当前 13 基团的简化版只能作为起步, 不能直接用于最终模型。

**验证方法**:
1. 对 Bicerano 304 数据集计算 GC_Tg
2. 与实验 Tg 计算 r²: 预期 > 0.85 (25+ 基团版)
3. 覆盖率统计: 预期 > 75%
4. 如果 r² < 0.7 → Yg 值有误, 从 Cao 2020 SI 重新提取

**计算成本**: 毫秒级 (纯 SMARTS 匹配)

### 2.4 3D 自由体积分数 — 用 Van Krevelen 公式 + 精确 V_vdW

**物理依据**: 自由体积理论说 Tg 发生在 f_g ≈ 0.025 时。

**现有实现问题** (`physical_proxy.py` 第 210-215 行):
```python
v_vdw = Descriptors.LabuteASA(mol)  # ← 这是表面积, 不是体积!
v_m = max(mw, 1.0)                  # ← 分子量 ≠ 摩尔体积!
vf = 1.0 - 1.3 * v_vdw / v_m        # ← 单位不对, 物理无意义
```
这个计算是**错误的**——LabuteASA 是表面积 (Å²)，mw 是分子量 (g/mol)，两者的比值没有物理意义。

**修正实现**:

```python
def compute_free_volume_corrected(smiles):
    """
    修正的自由体积估算。

    Van Krevelen 公式: f ≈ 1 - 1.3 × V_vdW / V_m
      V_vdW = van der Waals 体积 (cm³/mol)
      V_m   = 摩尔体积 (cm³/mol) = Mw / density

    问题: 密度未知。
    解决: 用 GC 法估算 V_m = ΣVwi / 0.6 (Van Krevelen 经验关系)
         0.6 是典型的非晶聚合物堆积分数
    """
    mol = Chem.MolFromSmiles(smiles.replace("*", "[H]"))
    if mol is None:
        return {'FV_fraction': np.nan, 'FV_packing': np.nan}

    # V_vdW 用 RDKit 3D 体积 (需要构象)
    mol_3d = Chem.AddHs(mol)
    params = AllChem.ETKDGv3()
    params.useRandomCoords = True
    if AllChem.EmbedMolecule(mol_3d, params) >= 0:
        AllChem.MMFFOptimizeMolecule(mol_3d)
        V_vdw = AllChem.ComputeMolVolume(mol_3d)  # Å³
    else:
        # 3D 嵌入失败 → 用 Labute ASA 近似 (不如 3D 精确但有值)
        # Labute ASA (Å²) → 体积近似: V ≈ (ASA)^1.5 / (6√π) (球体近似)
        asa = Descriptors.LabuteASA(mol)
        V_vdw = (asa ** 1.5) / (6 * np.sqrt(np.pi)) if asa > 0 else np.nan

    if V_vdw is None or np.isnan(V_vdw):
        return {'FV_fraction': np.nan, 'FV_packing': np.nan}

    # V_m 用 GC 法: 遍历 solubility_param.py 的 GC_GROUPS 累加 Vw
    # GC_GROUPS 已有 13 个基团的 Vw 值 (cm³/mol)
    from src.features.solubility_param import GC_GROUPS
    total_vw = 0.0
    for smarts_str, (name, ecoh, vw) in GC_GROUPS.items():
        pattern = Chem.MolFromSmarts(smarts_str)
        if pattern:
            matches = mol.GetSubstructMatches(pattern)
            total_vw += vw * len(matches)
    V_w_gc = total_vw  # cm³/mol
    if V_w_gc > 0:
        V_m = V_w_gc / 0.6  # cm³/mol (0.6 = 典型堆积分数)
    else:
        mw = Descriptors.MolWt(mol)
        V_m = mw / 1.1  # 密度 ≈ 1.1 g/cm³ 近似

    # 单位转换: V_vdw (Å³) → cm³/mol: ×6.022×10²³/10²⁴ = ×0.6022
    V_vdw_cm3 = V_vdw * 0.6022

    # Van Krevelen 自由体积分数
    if V_m <= 0:
        return {'FV_fraction': np.nan, 'FV_packing': np.nan}

    f = 1.0 - 1.3 * V_vdw_cm3 / V_m
    f = max(0.0, min(0.6, f))

    packing = V_vdw_cm3 / V_m

    return {
        'FV_fraction': f,        # 自由体积分数 (物理范围 0.02-0.5)
        'FV_packing':  packing,  # 堆积效率 (物理范围 0.5-0.75)
    }
```

**物理预测**: FV_fraction 与 Tg 应负相关 (r < -0.4)
**计算成本**: 需要 3D embedding，每分子 ~50ms
**与现有 Vf_estimate 的区别**: 单位正确、物理量正确

### 2.5 链刚性代理 C_n — 从构象采样估算

> **已实现**: N_eff 和 C_n 已合并为 `compute_3mer_physics(smiles)` (`src/features/chain_physics.py`)。一次 build_oligomer + EmbedMultipleConfs + MMFF94 能量计算，同时产出 8 维特征 (N_eff 4d + Cn 4d)。

**物理依据**: Adam-Gibbs CRR 大小与链刚性正相关。链刚性的经典度量是特征比 C_∞ = <R²>/(n·l²)。

**关键技术限制** (调研发现):
- C_n 收敛到 C_∞ 的速度是 **n^(-1/2)**，比预期慢
- PE (C_∞=6.7) 需要 n>10 键才达到 90%
- PS (C_∞~10) 需要 n>30-50
- **5-mer 不足以精确估算 C_∞**

**务实方案**: 不追求 C_∞ 的精确值，而是用 **C_n(3-mer) 作为链刚性的排序代理**。对 ML 来说，相对排序比绝对值更重要。

```python
def compute_chain_stiffness_proxy(smiles, n_confs=50):
    """
    从 3-mer 构象采样估算链刚性代理。

    不是 C_∞ 的精确值，而是 C_n(3-mer) 作为排序指标。
    物理保证: C_n(3-mer) 是 C_∞ 的单调函数 (Mattice 2004)
    → 作为 ML 特征只需要排序正确，不需要绝对值正确。
    """
    oligomer = build_oligomer(smiles, n=3)
    if oligomer is None:
        return {'Cn_proxy': np.nan, 'curl_ratio': np.nan,
                'curl_variance': np.nan, 'oligomer_level': 0}

    mol = Chem.MolFromSmiles(oligomer)
    if mol is None:
        return {'Cn_proxy': np.nan, 'curl_ratio': np.nan,
                'curl_variance': np.nan, 'oligomer_level': 0}

    # 关键: 在 AddHs 之前找链端 (AddHs 后 CH₃ degree 变成 4, 找不到了)
    mol_no_h = mol
    from rdkit.Chem import rdmolops
    dist_matrix_no_h = rdmolops.GetDistanceMatrix(mol_no_h)
    heavy_idx = [a.GetIdx() for a in mol_no_h.GetAtoms() if a.GetAtomicNum() > 1]

    if len(heavy_idx) < 4:
        return {'Cn_proxy': np.nan, 'curl_ratio': np.nan,
                'curl_variance': np.nan, 'oligomer_level': 0}

    # 直接用图直径找最远重原子对 (最稳健的方法)
    # 不依赖 degree==1 判断 (AddHs 后会失效)
    max_dist = 0
    first_atom, last_atom = heavy_idx[0], heavy_idx[-1]
    for i in heavy_idx:
        for j in heavy_idx:
            if i < j and dist_matrix_no_h[i][j] > max_dist:
                max_dist = dist_matrix_no_h[i][j]
                first_atom, last_atom = i, j

    n_backbone_bonds_estimate = int(max_dist)
    if n_backbone_bonds_estimate < 3:
        return {'Cn_proxy': np.nan, 'curl_ratio': np.nan,
                'curl_variance': np.nan, 'oligomer_level': 0}

    # 现在 AddHs 做 3D 嵌入
    mol = Chem.AddHs(mol)
    params = AllChem.ETKDGv3()
    params.useRandomCoords = True
    params.randomSeed = 42
    cids = AllChem.EmbedMultipleConfs(mol, numConfs=n_confs, params=params)

    if len(cids) < 5:
        return {'Cn_proxy': np.nan, 'curl_ratio': np.nan,
                'curl_variance': np.nan, 'oligomer_level': 0}

    # 优化构象
    for cid in cids:
        try:
            AllChem.MMFFOptimizeMolecule(mol, confId=cid, maxIters=200)
        except:
            pass

    # first_atom, last_atom 索引在 AddHs 后不变
    # (AddHs 在末尾追加 H 原子, 不改变原有原子的索引)
    n_backbone_bonds = n_backbone_bonds_estimate

    r_ee_sq = []
    for cid in cids:
        conf = mol.GetConformer(cid)
        p1 = conf.GetAtomPosition(first_atom)
        p2 = conf.GetAtomPosition(last_atom)
        r2 = (p1.x-p2.x)**2 + (p1.y-p2.y)**2 + (p1.z-p2.z)**2
        r_ee_sq.append(r2)

    # C_n 近似: <R²> / (n × l²)
    # 用实际键长而非固定 1.54 Å: C-O=1.43, Si-O=1.64, C-N=1.47
    # 从任意一个构象直接读取两端之间最短路径的真实 3D 键长
    conf0 = mol.GetConformer(cids[0])
    path = rdmolops.GetShortestPath(mol, first_atom, last_atom)
    bond_lengths_sq = []
    for k in range(len(path) - 1):
        p_a = conf0.GetAtomPosition(path[k])
        p_b = conf0.GetAtomPosition(path[k + 1])
        bl2 = (p_a.x-p_b.x)**2 + (p_a.y-p_b.y)**2 + (p_a.z-p_b.z)**2
        bond_lengths_sq.append(bl2)
    mean_l_sq = np.mean(bond_lengths_sq) if bond_lengths_sq else 1.54**2
    n_path_bonds = len(bond_lengths_sq) if bond_lengths_sq else n_backbone_bonds

    Cn = np.mean(r_ee_sq) / (n_path_bonds * mean_l_sq)

    # 链卷曲度: 实际末端距 / 完全伸展长度
    L_contour = sum(np.sqrt(bl2) for bl2 in bond_lengths_sq) if bond_lengths_sq else n_backbone_bonds * 1.54
    r_ee_mean = np.sqrt(np.mean(r_ee_sq))
    curl = r_ee_mean / L_contour  # 0=完全卷, 1=完全伸

    return {
        'Cn_proxy':       Cn,                                # 特征比近似
        'curl_ratio':     curl,                              # 链卷曲度
        'curl_variance':  np.std(np.sqrt(r_ee_sq)) / r_ee_mean,  # 构象可塑性
        'oligomer_level': 3,                                 # 元特征: 3-mer 成功
    }
```

**特征维度**: 4 (Cn_proxy, curl_ratio, curl_variance, oligomer_level)

**物理预测**: Cn_proxy 与 Tg 正相关 (刚性链 → C_n 大 → Tg 高)

**验证**:
- PE: Cn(3-mer) 应 ~3-4 (文献 C_∞=6.7, 3-mer 约为一半)
- PS: Cn(3-mer) 应 ~5-7 (文献 C_∞~10)
- PDMS: Cn(3-mer) 应 ~3-4 (文献 C_∞=6.3)
- 排序应为: PS > PE ≈ PDMS，与 Tg 排序 (PS > PE > PDMS) 一致

**3-mer + 5-mer 并行策略**: 所有分子先算 3-mer，再尝试 5-mer。5-mer 成功的分子用 5-mer 结果覆盖（更准），失败的保留 3-mer 结果。oligomer_level 标记每个分子的实际寡聚体长度 (0/3/5)，让模型知道数据质量。

### 2.7 GNN 预训练嵌入 — 周期性结构编码

**物理依据**: 聚合物是重复单元的周期性排列。当前所有特征都把重复单元当孤立分子，丢失了周期性信息。

**已有基础设施**:
- `src/gnn/physics_gat.py`: PhysicsGAT (3 层 GAT + GRIN 池化, ~28K 参数)
- `src/gnn/pretrainer.py`: 两阶段训练器 (预训练 → 微调)
- `src/gnn/graph_builder.py`: SMILES → PyG Data

**预训练数据** (去重后 ~10-15K, **不含目标数据集**):
- neurips_opp_tg.csv: 7,367 条
- polymetrix_tg.csv: 7,367 条
- ⚠️ **不用 unified_tg.parquet 做预训练**: 它是 Nested CV 的评估数据集。
  如果预训练中用了它的 Tg 标签，GNN 嵌入会隐含目标泄漏，
  导致 Nested CV R² 虚高——这恰好是我们要避免的。
  预训练只用外部数据，微调也只在 Nested CV 内层 fold 的训练集上做。

**物理增强** (在 graph_builder 中添加):
```python
# 在原子特征中加入:
'group_yg': Van Krevelen 基团贡献值 Yg (该原子所属基团)
'is_backbone': 是否主链原子 (0/1)
'is_junction': 是否聚合连接点邻近原子 (0/1)
'rotational_barrier': 该原子参与的最高旋转势垒 (从 RBP 查找表)

# 在边特征中加入:
'flexibility_weight': 该键的柔性权重 (从 _count_flexible_bonds_v2)
```

**训练方案**:
```
Stage 1: 预训练 (A800, ~4h)
  数据: ~10-15K 外部聚合物 (NeurIPS + PolyMetriX, 不含 unified_tg)
  任务: 预测 Tg (回归)
  架构: PhysicsGAT (已有, ~28K 参数)
  参数: lr=1e-3, dropout=0.2, epochs=200, batch=256

Stage 2: 微调 + 嵌入提取 (在 Nested CV 每个 outer fold 内做)
  在每个 outer fold 内:
    1. 用该 fold 的训练集 (~5989 条) 微调最后层 + 池化 + MLP
    2. 冻结前 2 层 GAT (保留预训练学到的通用表示)
    3. 微调完成后, 对全量 7486 条提取 64d 嵌入
    4. 该 fold 的后续模型用这份嵌入
  参数: lr=1e-4, dropout=0.3, epochs=50

  这样做的原因:
    - 微调让嵌入针对我们的 Tg 任务优化, 比通用嵌入信息量更大
    - 每个 fold 独立微调, 测试折的 Tg 标签从未被 GNN 见过, 零泄漏
    - 计算量: 15 folds × ~5 分钟/fold = ~1.5h (A800), 可接受
```

**计算成本**: A800 约 5.5 小时 (预训练 4h + 15 fold 微调 1.5h)

### 3.2 GC 残差学习 — 两种策略

之前的 "分子级混合目标" 方案有严重问题：残差 (-80~+80K) 和绝对 Tg (130~770K) 在同一个 y 空间里，模型无法区分这两种尺度，会混乱。

**正确做法：不在训练目标中混合，而是在集成层面处理。**

```
策略 A (推荐，简单有效):
  GC_Tg 作为特征输入，训练目标始终是绝对 Tg。
  模型自己学 "GC_Tg 和 Tg 的关系"，本质上隐式地做了残差学习。
  无覆盖的分子 GC_Tg = NaN，CatBoost 自动处理。

策略 B (进阶，需要额外模型):
  只在 GC 覆盖 >50% 的子集上训练一个专门的残差模型。
  目标 = Tg - GC_Tg (范围 -80~+80K，全部统一尺度)。
  无覆盖的分子不参与这个模型。
  这个模型作为物理专家委员会中的一个专家 (专家 3)。

策略 C (两个模型集成):
  模型 A: 全量数据，目标 = Tg，特征含 GC_Tg (NaN for 无覆盖)
  模型 B: GC 子集，目标 = Tg - GC_Tg，特征不含 GC_Tg
  元学习器学习何时信任 A vs B

推荐: 先用策略 A (零额外复杂度)，在 Phase E 尝试策略 B 作为专家 3。
```

**关键原则**: 永远不在同一个模型的训练目标中混合不同尺度的值。

### 质量红线 (只有这些情况才停下来解决)

```
❌ 代码 bug: 新特征的物理方向与理论预测完全相反且排查后确认不是数据问题
❌ 数据损坏: 特征矩阵中 NaN 比例 > 80% (说明计算流程有系统性错误)
❌ 基础设施: GNN 预训练 loss 不收敛 (模型架构或数据有根本问题)

⚠️ 这些不是红线 (记录但继续):
  · 某个特征 SHAP 排名低 → 可能在集成中有间接贡献
  · 单步 R² 没提升 → 最终集成才是判断标准
  · 5-mer 失败率 40% → 60% 有效数据仍有价值
  · MMFF 对特定分子类型不准 → 标记，让模型降权
```

## 六、预期与策略

### 可控因素 vs 不可控因素

| 因素 | 可控? | 影响 | 本方案的应对 |
|------|-------|------|------------|
| 特征工程质量 | ✅ | 核心杠杆 | 5 个物理尺度全覆盖 |
| 模型架构 | ✅ | 放大特征价值 | 物理专家委员会 |
| 数据清洗 | ✅ | 降低噪声底 | Phase A 物理规则清洗 |
| 测量噪声 σ_meas | ❌ 数据固有 | 决定天花板 | 量化它，但不被它限制 |
| 分子量/立构缺失 | ❌ 数据固有 | 3-5% 变异 | 接受，不试图编码不存在的信息 |
| MMFF 力场精度 | ⚠️ 可标记 | 影响 3D 特征质量 | NaN 标记不准的分子，模型自动降权 |
| 部分分子计算失败 | ✅ 可处理 | 特征矩阵有 NaN | CatBoost/TabPFN 原生支持 NaN |

### R² 目标的态度

- **目标 0.95 是方向，不是死线**
- 不预设每步增量，让实验数据说话
- Phase A 的天花板估算是参考信息，不是决策门槛
- **即使最终 R² = 0.93 而非 0.95，这套多尺度物理特征体系本身就是论文核心创新**
- 每个物理特征与 Tg 理论的对应关系、消融分析、有效/无效的解释——这些比一个数字更有学术价值

### 核苷酸迁移策略

核苷酸不是聚合物，很多新特征 (VPD, N_eff, Cn) 对它们物理上无意义。

策略：**分场景特征集**
- 通用聚合物 Tg: 全量多尺度特征 (目标 R² > 0.93)
- 核苷酸迁移: 保留 L1H 34d 特征集 (已验证 ATP 4.7K, ADP 0.4K)
- 两条线并行，互不干扰
