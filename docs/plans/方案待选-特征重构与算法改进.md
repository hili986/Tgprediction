## 二、特征工程重构方案

### 方案 F1：旋转势垒加权柔性 (RBP) — 替换 FlexibilityIndex

**物理依据**：FlexibilityIndex 计数旋转键但不区分势垒。C-C (14 kJ/mol)、C-O (5 kJ/mol)、Si-O (1 kJ/mol) 被等权对待。

**具体变更**：

```
现有: FlexibilityIndex = NumRotatableBonds / HeavyAtomCount

替换为 4 个 RBP 特征:
  RBP_weighted_flex   = Σ(1/barrier_i) / HeavyAtomCount   ← 加权柔性
  RBP_mean_barrier    = mean(barrier_i)                    ← 平均势垒
  RBP_max_barrier     = max(barrier_i)                     ← 瓶颈势垒
  RBP_barrier_diversity = std(barrier_i) / mean(barrier_i) ← 势垒多样性
```

**旋转势垒查找表** (从文献/量化计算):

| 键类型 | 势垒 (kJ/mol) | 来源 |
|--------|--------------|------|
| C(sp3)-C(sp3) | 14.0 | 乙烷实测 |
| C(sp3)-O | 5.0 | 二甲醚 |
| C(sp3)-N | 12.0 | 甲基胺 |
| Si-O | 1.0 | PDMS 理论 |
| C(sp2)-N (酰胺) | 80.0 | 共振半双键 |
| C(sp3)-C(aromatic) | 10.0 | 甲苯 |
| C(aromatic)-C(aromatic) | 20.0 | 联苯 |
| C(sp3)-S | 8.0 | 硫醚 |
| C-C 与大侧基相邻 | 18.0 | 位阻增加 |

**实施复杂度**: 低（纯 2D 计算，~30 秒/全数据集）
**预期效果**: R² +0.3-1.0%（精细化最强预测因子）

---

### 方案 F2：构象熵描述符 (CES) — 新增 3D 信息

**物理依据**：Gibbs-DiMarzio 理论的核心变量（构象熵 S_c）从未被直接编码。

**具体变更**：新增 5 维

```
CES_diversity        ← 多构象 RMSD 分散度 (高 → 低 Tg)
CES_energy_spread    ← 构象能量标准差 (高 → 复杂能量景观)
CES_gyration_var     ← 回转半径变异系数 (高 → 链可塑性)
CES_volume_ratio     ← vdW 体积/包络体积 (高 → 堆积紧 → 高 Tg)
CES_max_dihedral_range ← 主链二面角采样范围 (大 → 高柔性)
```

**计算方法**: RDKit ETKDGv3 生成 50 个 3D 构象 → MMFF 优化 → 统计分析
**实施复杂度**: 中高（~5h/全数据集，可并行）
**预期效果**: R² +0.5-2.0%（引入全新信息维度）

---

### 方案 F3：GC 残差学习 — 改变学习目标

**物理依据**：Van Krevelen GC 法 R²=0.99 (Liu et al. 2020)，ML 从零学习 Tg 浪费了 GC 已编码的信息。

**具体变更**：

```
现有: model.fit(X, Tg)           → 学习 Tg 全量
改为: model.fit(X, Tg - Tg_GC)   → 只学习 GC 无法解释的残差

最终预测: Tg_pred = Tg_GC + model.predict(X)
```

**GC 计算实现** — 两种策略:

| 策略 | 方法 | 覆盖率 | 精度 |
|------|------|--------|------|
| **策略 a** | RDKit BRICS 分解 + 58 基团查表 | ~70-80% | R²~0.95 |
| **策略 b** | Tg_GC 作为第 47 个特征输入（不改目标） | 100% | — |
| **策略 c** | 两阶段: 先用 GC 可覆盖的子集训残差，再全集训标准模型，集成 | 100% | 综合最优 |

**推荐策略 b（最简单有效）**：将 GC 预测值作为额外特征加入，让模型自动学习"何时信任 GC"。

**实施复杂度**: 策略 b 低 / 策略 a 中 / 策略 c 中高
**预期效果**: R² +0.5-2.0%（物理锚点 + 残差缩小）

---

### 方案 F4：H-bond 特征重构 — 精简 + 聚合

**物理依据**：10 个 SMARTS 计数全被 SHAP 淘汰，说明过于精细。但分子间力贡献 26% SHAP，不应丢弃。

**具体变更**：

```
现有 HBond 15 维:
  10 个 SMARTS 计数 (全被淘汰) + 4 个密度 + 1 个 CED

重构为 5 维:
  ced_weighted_sum        ← 保留 (SHAP #7, 6.89)
  hbond_network_density   ← 新: (供体×受体) / 重原子² (氢键网络潜力)
  polar_surface_fraction  ← 新: TPSA / MolWt (质量归一化极性)
  strong_weak_ratio       ← 新: 强氢键计数 / 总氢键计数
  interaction_diversity   ← 新: 不同类型氢键基团数 (种类多样性)
```

**效果**: 从 15 维减少到 5 维，消除噪声特征，保留物理信息

---

### 方案 F5：VPD 升级 — 5-mer + 构象采样

**物理依据**：当前 3-mer VPD 只捕捉近邻效应。链段运动涉及 10-50 个主链原子（≈3-15 个重复单元），3-mer 是下限。

**具体变更**：

```
现有: 3-mer 单构象 → 12 维
升级:
  VPD_5mer_*     ← 5-mer 标准化描述符 (更长链段, 6 维)
  VPD_conf_*     ← 3-mer 多构象统计 (3 构象的描述符方差, 3 维)
  VPD_chain_curl ← 5-mer 的 end-to-end distance / contour length (链卷曲度, 1 维)
```

**计算方法**:
- 5-mer 构建: 复用 `build_oligomer(smiles, n=5)`
- 多构象: ETKDGv3 生成 3 个构象，取统计量
- 链卷曲度: 首尾原子距离 / 主链键长总和

**实施复杂度**: 中（5-mer 构建可能对某些 SMILES 失败）
**预期效果**: R² +0.2-1.0%（更丰富的链段信息）

## 三、算法/模型改进方案

### 方案 M1：物理约束 CatBoost — 单调性 + 边界

**原理**：Phase 4 的 ConstrainedGBR 在 304 条上无增益，但在 7,486 条上约束可能有效。

**具体变更**：

```python
CatBoostRegressor(
    monotone_constraints={
        'RBP_weighted_flex': -1,      # 柔性↑ → Tg↓ (三大理论)
        'ced_weighted_sum':   1,      # CED↑ → Tg↑ (Van Krevelen)
        'L1_FractionCSP3':  -1,      # sp3↑ → 柔性↑ → Tg↓
        'VPD_RingCount_per_RU': 1,   # 环多 → 刚性 → Tg↑
    },
)

# 后处理: 物理边界裁剪
y_pred = np.clip(y_pred, 100, 700)  # K
```

**预期效果**: R² +0.1-0.5%（物理一致性提升 + 外推改善）
**风险**: 约束过强可能降低拟合灵活性

---

### 方案 M2：两阶段 Stacking — 异质基模型

**原理**：Phase 5 Stacking 收益小 (+0.4%)，因为基模型全是树模型（同质）。引入异质基模型增加多样性。

**具体变更**：

```
基模型层 (异质):
  ├─ CatBoost         ← 非线性、特征交互
  ├─ TabPFN v2        ← meta-learned prior
  ├─ Ridge Regression ← 线性基线
  ├─ KNN (k=10-20)    ← 局部模式
  └─ ElasticNet       ← 稀疏线性

元学习器: Ridge (L2 正则化)

两阶段:
  Stage 1: 各基模型在 inner CV 上生成 out-of-fold 预测
  Stage 2: 元学习器学习最优组合权重
```

**与 Phase 5 Stacking 的区别**:
- Phase 5: CatBoost + LightGBM + ET + XGBoost → Ridge (同质树模型)
- 本方案: CatBoost + TabPFN + Ridge + KNN + ElasticNet → Ridge (异质模型)

**预期效果**: R² +0.5-1.5%（模型多样性增加 → 元学习器有更多"材料"可组合）

---

### 方案 M3：残差预测误差分析驱动的特征补充

**原理**：分析 TabPFN/CatBoost 预测误差最大的分子，找出它们的共同特点 → 针对性补充特征。

**具体操作**：

```
Step 1: 用当前最优模型 (TabPFN) 对全数据集做 out-of-fold 预测
Step 2: 找出 |error| > 50K 的分子 (~5-10%)
Step 3: 分析这些高误差分子的共同特征:
  - 含什么特殊基团？(硅氧烷、含氟、高度交联等)
  - SHAP 哪个特征贡献了最大误差？
  - 是否集中在某个 Tg 区间？
Step 4: 针对性设计新特征或训练专用子模型

可能发现:
  - 高误差集中在含 Si-O 的聚合物 → 添加 Si-O 键计数特征
  - 高误差集中在 Tg > 500K → 高温区需要不同的物理模型
  - 高误差集中在多环分子 → 环刚性编码不足
```

**预期效果**: 定向修复最大误差源，R² +0.3-1.0%
**额外价值**: 误差分析本身是论文的重要内容

---

### 方案 M4：特征选择优化 — 物理引导的 Top-K

**原理**：当前 SHAP Top 15 是数据驱动选择。改为**物理引导**——确保每个物理维度至少有代表。

**具体变更**：

```
数据驱动 Top 15 (现有):
  可能出现: 4 个柔性特征 + 3 个环特征 + 0 个极性特征
  → 某个物理维度可能被完全丢弃

物理引导 Top-K:
  Step 1: 定义 4 个物理维度 (柔性/刚性/分子间力/尺寸)
  Step 2: 每个维度强制保留 SHAP 最高的 2-3 个特征
  Step 3: 剩余名额由全局 SHAP 排名填充
  Step 4: 总维度控制在 12-18

物理引导 Top 14 示例:
  柔性 (4): FlexibilityIndex, VPD_junction_flex, FractionCSP3, VPD_RotBonds_delta
  刚性 (3): VPD_RingCount_per_RU, L1_RingCount, BalabanJ
  分子间力 (4): ced_weighted_sum, NumHDonors, TPSA, VPD_LogP_delta
  溶解度 (1): SOL
  VPD 独有 (2): VPD_TPSA_per_RU, VPD_RotBonds_per_RU
```

**预期效果**: R² 与数据驱动 Top 15 持平或略优，但物理完备性更好

## 四、综合方案对比

### 方案 A：保守改进（低风险，快速实施）

```
特征: F1 (RBP) + F4 (H-bond 精简) = 39 维
模型: CatBoost + 物理单调约束 (M1)
预期: R² ≈ 0.878-0.885 (CatBoost), 0.900-0.905 (TabPFN)
工作量: 1-2 天
```

**优势**: 风险最低、代码改动最小、可快速验证
**劣势**: 创新性有限，可能不足以突破 0.90

---

### 方案 B：物理增强（中等风险，完整物理信息）

```
特征: F1 (RBP) + F2 (CES) + F3b (GC 特征) + F4 (H-bond 精简) = 45 维
模型: 物理约束 CatBoost (M1) + 异质 Stacking (M2) + 误差分析 (M3)
预期: R² ≈ 0.888-0.900 (CatBoost), 0.905-0.915 (Stacking/TabPFN)
工作量: 1-2 周
```

**优势**: 每个改进有明确物理依据、创新性强（论文材料丰富）
**劣势**: CES 计算耗时、GC 分解可能覆盖率不足

---

### 方案 C：全面重构（高风险，最大潜力）

```
特征: F1 + F2 + F3a (GC 残差) + F4 + F5 (VPD 5-mer) = 55 维
模型: M1 + M2 + M3 + M4 (物理引导特征选择)
  → 先 GC 残差学习 → 物理约束 CatBoost → 异质 Stacking → 误差驱动迭代
预期: R² ≈ 0.895-0.910 (CatBoost), 0.910-0.920 (Stacking)
工作量: 2-4 周
```

**优势**: 将所有物理信息注入模型、有望达到 0.91+ R²
**劣势**: 实施复杂、GC 残差学习可能在某些聚合物上失败、5-mer 构建可能有大量失败

## 五、推荐方案与实施路线

### 推荐: 方案 B (物理增强) — 渐进实施

**理由**:
1. 方案 A 太保守，可能不足以突破 0.90
2. 方案 C 风险过高，尤其是 GC 残差学习和 5-mer VPD 都有较大不确定性
3. 方案 B 兼顾创新性和可控性，每一步都有明确的物理依据和回退方案

### 渐进实施路线

```
第 1 步: F1 (RBP) + F4 (H-bond 精简)        ← 1 天, 零风险
  ├─ 实现旋转势垒查找表
  ├─ 替换 FlexibilityIndex 为 RBP_weighted_flex
  ├─ 精简 HBond 15→5 维
  ├─ 跑 CatBoost Nested CV 对比基线

  ├─ 实现 Van Krevelen 基团分解 (BRICS/SMARTS)
  ├─ 计算 Tg_GC 作为第 N+1 个特征
  ├─ 跑对比实验: 有/无 Tg_GC 的 R² 差异
  └─ 验证: Tg_GC 的 SHAP 排名？GC 覆盖率？

第 3 步: M1 (物理约束 CatBoost)              ← 半天
  ├─ 对 4 个物理量添加单调约束
  ├─ 添加边界裁剪 [100, 700]K
  └─ 验证: 约束是否改善外推？核苷酸迁移是否受益？

第 4 步: F2 (CES 构象熵)                     ← 2-3 天, 中等风险
  ├─ 对 7486 分子做 3D 构象采样 (50 conf/mol)
  ├─ 计算 5 维 CES 特征
  ├─ 合并到特征集，跑对比
  └─ 验证: CES_diversity 是否提供独立信息？

第 5 步: M2 (异质 Stacking) + M3 (误差分析)  ← 2-3 天
  ├─ 构建 5 基模型异质 Stacking
  ├─ 分析 |error| > 50K 的分子特征
  ├─ 针对性补充特征或子模型
  └─ 验证: 最终 R² 和误差分布

每步检查点:
  ✓ R² 是否提升？(否 → 回退该步)
  ✓ SHAP 排名是否合理？(新特征应进 Top 15)
  ✓ 核苷酸迁移是否受损？(是 → 保留分场景特征集)
```

## 七、风险与缓解

| 风险 | 概率 | 影响 | 缓解措施 |
