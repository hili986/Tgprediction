## 1. 目标

从 3-mer 寡聚体的 3D 构象采样中提取 **8 维链段物理特征**，编码 Gibbs-DiMarzio 和 Adam-Gibbs 理论的核心变量。这些是当前 PHY 48d 特征集**无法覆盖的新信息维度**——从原子/键/重复单元尺度跃迁到链段尺度。

## 2. 物理依据

| 特征 | 物理理论 | 物理含义 | 预期与 Tg 关系 |
|------|---------|---------|---------------|
| Neff_300K | Gibbs-DiMarzio S_c | 300K 下有效构象数 (Boltzmann 加权) | 负相关 (柔→多构象→低 Tg) |
| Neff_500K | Gibbs-DiMarzio S_c | 500K 下有效构象数 | 负相关 |
| Neff_ratio | Angell fragility | Neff_500K/Neff_300K, 能量景观陡峭度 | >2 → fragile 型 |
| conf_strain | 构象应变 | 平均能 - 最低能 (kcal/mol) | 正相关 (受限→高 Tg) |
| Cn_proxy | Adam-Gibbs CRR | 特征比近似 <R²>/(n·l²) | 正相关 (刚→大 Cn→高 Tg) |
| curl_ratio | 链卷曲度 | 末端距/轮廓长 (0=卷,1=伸) | 负相关 (伸展→刚→高 Tg) |
| curl_variance | 构象可塑性 | std(R_ee)/mean(R_ee) | 正相关 (多变→柔→低 Tg) |
| oligomer_level | 元特征 | 0=失败, 3=3-mer 成功 | 告诉模型数据质量 |

### 3.2 能量后端 — 双后端自动切换

```
_get_backend() 自动检测:
  有 CUDA + aimnet2calc → AIMNet2 GPU (MAE=0.32 kcal/mol vs DFT)
  否则 → MMFF94 CPU (RDKit 内置)
  AIMNet2 对特定分子失败 → 逐分子 fallback 到 MMFF
```

**实际运行决策**: 使用 MMFF94 纯 CPU 16 核并行。原因:
- 瓶颈是 ETKDGv3 构象生成 (CPU)，不是能量计算
- AIMNet2 + joblib 多进程会导致 GPU 竞争 (每个 worker 独立加载模型)
- MMFF94 的能量排序对 Boltzmann N_eff 足够 (我们要相对排序，不要绝对精度)
- 纯 CPU 16 核无冲突，稳定可靠

**如何强制 MMFF**: `CUDA_VISIBLE_DEVICES="" python scripts/compute_chain_physics.py`

## 4. 当前状态 — 全量计算

### 4.1 运行命令

```bash
# A800 服务器上 (302-GPU2)
nohup env CUDA_VISIBLE_DEVICES="" python scripts/compute_chain_physics.py \
    --n-confs 50 --n-jobs 16 > chain_physics.log 2>&1 &
```

### 4.2 服务器配置

- Intel Xeon Silver 4208 × 2P = 16 物理核 @ 2.10GHz
- `--n-jobs 16` (物理核数，超线程对计算密集无收益)
- `CUDA_VISIBLE_DEVICES=""` 强制 MMFF CPU 后端

### 4.3 Checkpoint 机制

- 每 100 条保存到 `data/chain_physics_checkpoint.json`
- 中断后重新运行同一命令 (不加 `--reset`) 自动续传
- 加 `--reset` 从头开始
- 完成后输出到 `data/chain_physics_features.parquet`
- checkpoint 文件在全部完成后自动删除

### 4.4 监控命令

```bash
tail -f chain_physics.log           # 实时进度
tail -1 chain_physics.log           # 最新一行
wc -c data/chain_physics_checkpoint.json  # checkpoint 文件大小 (增长=在跑)
ps aux | grep python | grep -v grep      # 进程状态
```

### 4.5 预计耗时

- 基准: ~60s/mol (单核, 含 ETKDGv3 + MMFF)
- 16 核并行: ~60/16 ≈ 4s/mol (理论)
- 7486 条: ~8-10 小时 (含复杂大分子的长尾)
- checkpoint 每 100 条 ≈ 每 7 分钟保存一次

## 6. 已知风险

| 风险 | 影响 | 缓解 |
|------|------|------|
| 大分子 3-mer (>100 重原子) ETKDGv3 极慢 | 长尾效应, 个别分子 >10min | NaN + oligomer_level=0 |
| MMFF94 芳基酰胺势垒过高 | N_eff 排序可能偏 | 作为 ML 特征仍可用 |
| 3-mer 太短无法收敛到 C_∞ | Cn_proxy 绝对值不准 | 排序正确即可 (Mattice 2004) |
| PET 类芳香酯 N_eff 异常高 | Tg 由链间力决定, 非链柔性 | 模型从其他特征补偿 |
