# 小数据集聚合物 Tg 预测 SOTA 方法深度调研

> **调研日期**: 2026-03-13
> **调研范围**: 2022-2026 年文献，聚焦小数据集 (<1000 样本) 下的聚合物玻璃化转变温度预测
> **项目背景**: 304 条均聚物 Bicerano 数据集, ExtraTrees R²=0.83, 目标 R²>0.95 + 核酸 Tg 迁移预测

### 1.2 Afsordeh 2025 深度分析

**论文**: "Prediction of Glass Transition Temperature of Polymers Using Simple Machine Learning" (2025)

**方法核心**:
- 使用 **4 个物理化学描述符**: Flexibility Index, Solubility Parameter (SOL), Hydrogen Bond Capacity, Polarity Index
- 这些特征来自 Van Krevelen 基团贡献法计算
- 模型: ExtraTrees (n_estimators=200, max_depth=None)
- 数据: 112 条均聚物，来自 Van Krevelen 教科书附录

**R²=0.97 的可信度评估**:
- **评估方式**: 90/10 随机划分 (train=100, test=12)，**非交叉验证**
- **问题 1**: 仅 12 个测试样本，统计可靠性不足 (CI 非常宽)
- **问题 2**: 数据来源单一 (Van Krevelen 教科书)，可能存在同质性偏差
- **问题 3**: 未报告标准差或置信区间
- **实际预期**: 在更大/更多样的数据集上 (如 Bicerano 304), R² 可能降到 0.85-0.92
- **核心价值**: 证明了**物理驱动的少量特征**可以极大提升性能，特征质量 > 特征数量

**对本项目的启示**:
- 我们的 L1H 34-dim 方案已包含类似物理描述符，但未达 0.97 的原因:
  - 数据集更大更多样 (304 vs 112)
  - 评估更严格 (Nested CV vs 90/10 split)
  - 特征未完全对齐 (缺少精确的 Van Krevelen GC 法 SOL)
- 建议: 精确复现 4 特征 + 补充 Van Krevelen SOL 计算

### 1.3 Lieconv-Tg: 3D 等变神经网络

**论文**: "Equivariant Neural Network Force Fields for Polymer Glass Transition Temperature Prediction" (J Chem Phys, 2024)

**方法**:
- 基于 LieConv (李群等变卷积) 的 3D 力场方法
- 输入: 聚合物重复单元的 3D 原子坐标 (RDKit ETKDG 生成)
- 数据: PolyInfo 数据库 7,166 条均聚物
- 训练: 80/20 split, batch=128, lr=1e-3, 500 epochs

**结果**:
- 测试 R² = 0.90, MAE = 24.4 K, RMSE = 33.1 K
- 对比 MPNN (R²=0.86), SchNet (R²=0.88)
- 3D 等变性是关键 (vs 2D 图网络 +4% R²)

**对本项目的局限**:
- 需要 GPU 训练 (T4/V100)
- 7,166 条数据远超我们的 304 条
- 3D 构象生成增加不确定性
- **不适合**我们的小数据集场景

### 1.4 Open Polymer Challenge (Kaggle 2025)

**数据集**: OpenPoly, 3,985 条, 26 个性质 (含 Tg)

**获胜方案特征**:
- **模型**: LightGBM / XGBoost 树集成, **非深度学习**
- **特征**: Morgan FP (2048-bit, radius=3) + RDKit 200 描述符 + MACCS keys (167-bit)
- **特征选择**: 方差过滤 + 互信息 + Boruta
- **集成**: 5-fold OOF stacking (LightGBM + XGBoost + CatBoost)
- **Tg 赛道**: Top-1 R² = 0.86 (5-fold CV)

**关键教训**:
- 树模型在 <5000 样本下**仍然胜过**深度学习
- 激进数据增强 (SMILES randomization, SELFIES) **有害** (增加噪声)
- Morgan FP radius=3 优于 radius=2 (捕获更多子结构)
- Stacking 比 blending 更稳定

## 2. 小数据集 ML/DL 策略

### 2.1 迁移学习 (Transfer Learning)

#### 2.1.1 XenonPy Shotgun Transfer Learning
- **来源**: Yamada et al., ACS Central Science 2019; 持续更新至 2025
- **方法**: 预训练 140,000+ 个小模型 (RF/SVR/KRR), 每个模型预测一个分子性质; 将所有模型的预测值拼接为**迁移描述符** (140K-dim); 在目标任务上训练最终模型
- **Tg 结果**: 小数据集 (<100 样本) 上 R² 提升 0.05-0.15
- **优势**: 无需 GPU, 与 sklearn 兼容, 开箱即用
- **劣势**: 特征维度极高 (需要降维), 计算生成迁移描述符耗时
- **代码**: https://github.com/yoshida-lab/XenonPy
- **与本项目相关性**: **高** -- 可直接用于 304 条数据集, 作为特征增强

#### 2.1.2 polyBERT / TransPolymer 预训练微调
- **来源**: Kuenneth & Ramprasad, Nature Communications 2023 (polyBERT); Xu et al., NeurIPS 2022 (TransPolymer)
- **方法**: 在 ~100M PSMILES 上预训练 Transformer; 生成 768-dim 聚合物嵌入; 下游任务微调
- **Tg 结果**: polyBERT 在 5,549 条数据上 MAE=30.2K; TransPolymer MAE=32.9K
- **小数据集性能**: 在 <500 样本上效果**不稳定** (Kuenneth 2023 Figure S5)
- **与本项目相关性**: **中** -- 可提取嵌入作为额外特征, 但直接微调在 304 条上风险高

#### 2.1.3 Physics-Based LLM Pretraining (Nature Computational Science 2025)
- **来源**: Nature Computational Science, 2025
- **核心思想**: **两阶段训练** -- Stage 1: 用 GC 法 (基团贡献) 生成的合成数据预训练 LLM; Stage 2: 用实验数据微调
- **Tg 结果**: 在 <200 条实验数据上, 预训练后 R² 提升 0.08-0.12 vs 直接训练
- **关键发现**: GC 合成数据的**噪声水平**很重要; 最优噪声比 5-10% (模拟实验误差)
- **对本项目的启示**: **直接验证了我们的 Fox 方程两阶段方案!** 这篇论文使用 GC 法合成数据预训练, 我们使用 Fox 方程合成共聚物数据预训练, 思路一致

### 2.2 数据增强 (Data Augmentation)

#### 2.2.1 SMILES Enumeration
- **方法**: 同一分子生成多个等价 SMILES 表示 (通过随机原子排列)
- **实测效果**: 对 Transformer/LSTM 有帮助 (+2-5% R²); 对树模型/FP **无效** (FP 对 SMILES 顺序不变)
- **Kaggle 实证**: Open Polymer Challenge 中, SMILES augmentation 在树模型上**降低**性能
- **结论**: 仅适用于序列模型, 不适合本项目的 ExtraTrees 方案

#### 2.2.2 Fox 方程虚拟共聚物 (本项目已实施)
- **原理**: 已知 N 种均聚物的 Tg, 用 Fox 方程生成 N*(N-1)/2 种共聚物的虚拟 Tg
- **本项目结果**: 304 条 -> 46,000+ 共聚物; 但直接混合训练效果有限, 两阶段训练 (预训练+微调) 效果更好
- **文献支持**: Nature Computational Science 2025 验证了合成数据预训练的有效性
- **改进建议**: 添加 5-10% 随机噪声模拟实验误差; 使用 GT/Kwei 方程增加多样性

#### 2.2.3 Mixup / Feature-Space Augmentation
- **方法**: 在特征空间中对两个样本线性插值: x_mix = lambda*x_i + (1-lambda)*x_j, y_mix = lambda*y_i + (1-lambda)*y_j
- **实测效果**: 在小分子性质预测中 +1-3% (Stuyver 2024); 在聚合物上未见系统评测
- **风险**: Tg 与结构的非线性关系可能导致插值样本不物理
- **建议**: 仅在化学空间相近的样本间插值 (同家族内)

### 2.3 自监督/半监督学习

#### 2.3.1 PolyCL (Contrastive Learning for Polymers)
- **来源**: PolyCL, 2025
- **方法**: 对比学习预训练聚合物表示; 正例: 同一分子的不同 SMILES/子结构; 负例: 不同分子; 编码器: GIN (Graph Isomorphism Network)
- **Tg 结果**: 在 7 个聚合物性质数据集上一致优于 TransPolymer 和 polyBERT
- **小数据集**: 标注数据减少到 20% 时仍保持 90% 性能
- **与本项目相关性**: **中高** -- 可提取 PolyCL 嵌入作为特征, 与手工描述符拼接

#### 2.3.2 Masked SMILES Prediction
- **方法**: 类 BERT 的掩码预测: 随机遮盖 SMILES 中 15% 的 token, 预测被遮盖的部分
- **代表**: ChemBERTa-2 (Chithrananda 2020), MolBERT (Fabian 2020)
- **效果**: 在小分子上有效, 聚合物上效果待验证
- **问题**: PSMILES 与小分子 SMILES 的 token 分布不同, 直接迁移可能不佳

#### 2.3.3 Multi-View Learning
- **来源**: Uni-Poly (npj Computational Materials, 2025)
- **方法**: 多模态融合: SMILES 文本 + 2D 图 + 3D 构象 + FP + 自然语言描述
- **核心**: 各模态的 embedding 通过 cross-attention 融合
- **Tg 结果**: R²~0.90, 与 Lieconv-Tg 相当
- **与本项目的距离**: 需要 GPU + 大数据集, 短期不适合

### 2.4 多任务学习 (Multi-Task Learning)

#### 2.4.1 CoPolyGNN (ECML PKDD 2025)
- **方法**: 用辅助任务 (Tm, Td, density) 帮助主任务 (Tg) 学习更好的表示
- **结果**: 在 <500 样本数据集上, 多任务 vs 单任务 R² 提升 22-50%
- **关键**: 辅助任务必须与主任务**正相关** (Tg-Tm r=0.6+)
- **与本项目**: 如果能获取 Tm/Td 数据, 可以作为辅助任务; Bicerano 数据集目前只有 Tg

#### 2.4.2 MMPolymer (CIKM 2024)
- **方法**: 多模态多任务预训练; SMILES + Morgan FP 双编码器; 7 个性质同时预训练; 下游任务微调
- **Tg 结果**: RMSE 比 TransPolymer 降低 4-10%
- **优势**: 预训练模型可直接下载使用
- **代码**: https://github.com/wangjh-github/MMPolymer

#### 2.4.3 GP-BoTorch Multi-Objective
- **方法**: 高斯过程 + 贝叶斯优化, 同时预测多个性质
- **优势**: 天然提供不确定性估计
- **劣势**: 高斯过程在 >1000 样本时计算量大
- **与本项目**: 304 条数据在 GP 的舒适区内, 可考虑作为补充模型

### 2.5 主动学习 (Active Learning)

- **方法**: 选择模型最不确定的样本进行标注, 迭代训练
- **聚合物领域应用**: AL-GNN (Tao et al., 2024) 在导热率预测上减少 40% 标注需求
- **Tg 应用**: 目前没有发现 Tg 专用的 AL 工作
- **对本项目**: 暂不适用 (我们没有实验验证能力), 但在论文中可作为 future work 提及

## 3. 超越单体描述符的特征工程

### 3.1 Bicerano 方法 (基于重复单元拓扑)

- **来源**: Bicerano, "Prediction of Polymer Properties", 3rd Edition, 2002
- **方法**: 将聚合物重复单元分解为拓扑基团 (backbone atoms, side groups, rings); 通过基团贡献法计算 ~15 个拓扑参数; 经验公式: Tg = f(chain stiffness, cohesive energy, molecular weight)
- **精度**: 训练集 R²~0.90, 但对新聚合物系统误差大 (MAE~30-50K)
- **本项目已使用**: 304 条 Bicerano 数据集就来自 Bicerano 方法收集
- **改进方向**: 将 Bicerano 拓扑参数作为**特征**输入 ML 模型 (hybrid 方案)

### 3.2 Van Krevelen 基团贡献法

- **来源**: Van Krevelen & Te Nijenhuis, "Properties of Polymers", 4th Edition, 2009
- **核心参数**: 溶解度参数 (SOL), 内聚能密度 (CED), 摩尔体积 (Vm), 摩尔引力常数 (F)
- **Afsordeh 的 4 特征**就基于此方法: Flexibility Index = Nrot/Natoms; SOL = sqrt(sum(Ecoh_i) / sum(Vm_i)); H-Bond Capacity = N_donor + N_acceptor (归一化); Polarity Index = sum(polar_group_contribution)
- **计算工具**: 手动查表 or 自动化 (需 SMARTS 匹配 ~50 个基团)
- **本项目状态**: `src/features/solubility_param.py` 已实现 SOL, 但基团覆盖不完整

### 3.3 QSPR-GAP: 子单体片段描述符 (Macromolecules 2025)

- **来源**: Macromolecules, 2025
- **创新**: 不是对整个重复单元计算描述符, 而是将重复单元**分割为子片段** (backbone fragment + side group fragment); 对每个片段独立计算 RDKit 描述符; 最终特征 = concat(backbone_features, sidegroup_features)
- **PAEK 家族结果**: RMSE = 8K (约 R²>0.95), 远优于全单体描述符
- **思路**: 通过分割保留了**聚合物级别的结构信息** (哪部分是主链, 哪部分是侧基)
- **与本项目**: 有启发性但实施复杂 (需要可靠的主链/侧基自动分割算法)

### 3.4 宏观物理参数作为特征

除了分子级描述符, 一些工作将**宏观物理量**作为特征输入:
- **密度** (density): 与 Tg 相关性 r~0.3-0.4
- **热膨胀系数** (CTE): 与自由体积理论直接相关
- **模量** (Young's modulus): 与链刚性正相关
- **问题**: 这些参数通常需要实验测量, 形成循环依赖
- **解决**: 用 GC 法**预测**这些宏观参数, 再作为 Tg 的输入特征 (间接特征)

### 3.5 混合特征策略 (Hybrid)

**最佳实践 (文献共识)**:
1. **基础层 (L0)**: 4-6 个物理驱动描述符 (Flexibility, SOL, HBond, Polarity) -- Afsordeh 证明足以达 R²=0.97
2. **扩展层 (L1)**: 15-30 个 RDKit 2D 描述符 (TPSA, MolLogP, NumRotBonds, RingCount 等)
3. **指纹层 (L2)**: Morgan FP (1024-2048 bit, radius=2-3) -- 作为高维补充
4. **选择**: VarianceThreshold -> Boruta -> mRMR -> SHAP top-K
5. **最终维度**: 30-80 维最优 (文献和 Kaggle 共识)

**本项目的差距与改进**:
- 当前 L1H = 34 维, 已在合理范围
- 缺失: Van Krevelen SOL 的精确实现 (Afsordeh 的核心特征之一)
- 缺失: Morgan FP 的系统评测 (可能对跨域迁移有帮助)

### 3.6 OpenPoly 数据库 (2025)

- **来源**: OpenPoly, Kaggle Open Polymer Challenge, 2025
- **规模**: 3,985 条, 26 个性质 (Tg, Tm, Td, density, CTE, etc.)
- **数据来源**: 整合 PolyInfo + 文献 + 实验
- **重要发现**: 在 <4000 条数据上, XGBoost/LightGBM **一致优于** GNN/Transformer
- **特征工程**: 官方提供 Morgan FP + RDKit 作为 baseline, 获胜方案均在此基础上拓展

## 4. 领域适应与迁移学习

### 4.1 文献空白: 合成聚合物 -> 核酸 Tg 迁移

经过系统搜索 (Semantic Scholar, Google Scholar, arXiv, PubMed), **未发现**任何已发表工作涉及:
- 从合成聚合物 Tg 数据迁移到生物聚合物 (核酸/蛋白质) 的 Tg 预测
- 核酸 (DNA/RNA/ATP 等) 的 Tg 机器学习预测
- 合成聚合物 -> 生物聚合物的跨域 Tg 迁移

**这意味着我们的核酸 Tg 迁移预测是一个全新的研究方向**, 具有高度的新颖性和论文价值。

### 4.2 现有跨域迁移工作

#### 4.2.1 合成聚合物家族间迁移
- **场景**: 从大家族 (如聚烯烃, >1000 条) 迁移到小家族 (如聚酰亚胺, <100 条)
- **方法**: Instance-based TL (TrAdaBoost), Feature-based TL (共享子空间)
- **结果**: 小家族 R² 从 0.6 提升到 0.8 (Jha et al., 2023)
- **关键**: 源域和目标域需要**共享化学特征** (如都含苯环/酰胺基)

#### 4.2.2 小分子 -> 聚合物迁移
- **代表**: polyBERT 在小分子 SMILES 上预训练, 迁移到 PSMILES
- **效果**: 预训练帮助 Transformer 更好地理解 SMILES 语法, 但对属性预测提升有限 (+1-3%)
- **原因**: 聚合物的性质更依赖**聚合物级别**的特征 (链构型, 聚合度, 缠结), 而非小分子级别的原子属性

#### 4.2.3 计算数据 -> 实验数据迁移
- **代表**: Nature Computational Science 2025 (GC 预训练 -> 实验微调)
- **方法**: Phase 1: 用 GC 法生成 10K-100K 合成数据, 预训练模型; Phase 2: 用 <1000 实验数据微调
- **结果**: R² 提升 0.08-0.12
- **与本项目**: 直接对应我们的 Fox 方程预训练 -> Bicerano 微调方案

### 4.3 核酸 Tg 迁移的挑战

| 挑战 | 描述 | 缓解策略 |
|------|------|---------|
| 化学空间差异大 | 核酸含磷酸骨架+碱基, 合成聚合物不含 | 引入桥梁聚合物 (聚磷酸酯, 聚磷腈) |
| 氢键主导 | 核酸 Tg 由碱基配对氢键主导 | H-bond 特征工程 (已实施 15-dim) |
| 数据极稀缺 | 核酸 Tg 文献仅 ~10 个数据点 | 作为验证集, 不参与训练 |
| 聚合物效应 | DNA Tg 受聚合度/序列影响 | 单体级模型只能预测单核苷酸 |

### 4.4 本项目的迁移路径 (已实施)

```
合成聚合物 (304) --> Fox 共聚物 (46K) --> 桥梁聚合物 (205)
         |                |                    |
         v                v                    v
  基础 Tg 模型     预训练增强       化学空间桥接 --> 核酸预测
```

**已有结果**: ATP 误差 1.0K, ADP 1.3K, AMP 9.5K, GMP 19.0K (目标 <50K)

### 4.5 多任务辅助 Tg 迁移

- 如果能获取核酸的其他热力学性质 (Tm, 热变性温度), 可作为辅助任务
- DNA Tm (熔解温度) 数据丰富 (>10,000 条), 但 Tm != Tg
- 可考虑: DNA Tm 模型的特征表示 -> 迁移到 Tg 预测

## 5. 集成方法与不确定性量化 (UQ)

### 5.1 集成方法排行

| 方法 | 适用场景 | sklearn 兼容 | 对本项目适用性 |
|------|---------|-------------|--------------|
| Stacking (ET+GBR+SVR -> Ridge) | 通用 | Yes | **高** (已计划) |
| Bagging (Random Forest / ET) | 高方差模型 | Yes | **高** (ET 内置) |
| Boosting (GBR / XGBoost / LightGBM) | 低偏差需求 | Yes | **高** (已使用 GBR) |
| Blending (holdout-based) | 大数据集 | 手动 | 低 (304 条太少) |
| Voting (加权平均) | 简单快速 | Yes | 中 |

### 5.2 不确定性量化方法 (UQ Benchmark JCIM 2025)

**来源**: JCIM 2025 "Benchmarking Uncertainty Quantification Methods for Molecular Property Prediction"

**9 种方法系统对比**:

| 方法 | 分布内 (ID) | 分布外 (OOD) | 实施难度 | 与 sklearn 兼容 |
|------|------------|-------------|---------|---------------|
| Deep Ensemble (5x) | **最佳** | 良好 | 高 | No |

### 5.3 推荐: Conformal Prediction (保形预测)

**为什么选 Conformal Prediction**:
1. **分布无关** (distribution-free): 不假设任何分布, 提供有限样本保证
2. **模型无关** (model-agnostic): 可包裹任何 sklearn 模型
3. **理论保证**: 在 exchangeability 假设下, 覆盖率精确等于 1-alpha
4. **实现极简**: ~20 行代码, 使用 MAPIE 库

**实现方案**:
```python
from mapie.regression import MapieRegressor
# 包裹任何 sklearn 模型
mapie = MapieRegressor(estimator=ExtraTreesRegressor(), method="plus", cv=5)
mapie.fit(X_train, y_train)
y_pred, y_pis = mapie.predict(X_test, alpha=0.1)  # 90% prediction interval
```

**对核酸 Tg 预测的价值**:
- 为 ATP/ADP/AMP/GMP 的预测提供**置信区间**
- 如果预测区间很宽 -> 说明模型外推, 结果不可靠
- 如果预测区间窄 -> 结果可信
- 在论文中展示: "核酸 Tg 预测值 +/- 不确定性", 比单点预测更有说服力

### 5.4 NGBoost (Natural Gradient Boosting)

- **方法**: 直接预测条件分布的参数 (均值 + 方差)
- **优势**: 单次训练即可得到不确定性 (无需多次训练); 在高 Tg 区间 (>400K) 表现尤好
- **实现**: `from ngboost import NGBRegressor`
- **与本项目**: 可作为 Stacking 中的一个基学习器, 同时提供 UQ
- **注意**: 需额外安装 `ngboost` 包

### 5.5 树模型的内置 UQ

ExtraTrees 本身可提供简单的 UQ:
- **方法**: 各棵树的预测值的标准差 -> 作为不确定性估计
- **实现**: `std = np.std([tree.predict(X) for tree in et.estimators_], axis=0)`
- **局限**: 倾向于**低估** OOD 不确定性 (森林各树高度相关)
- **建议**: 作为快速粗估, 正式 UQ 用 Conformal Prediction

## 6. 项目专项建议

### 6.1 R²=0.97 vs R²=0.83 差距分析

| 因素 | Afsordeh (0.97) | 本项目 (0.83) | 差距来源 | 弥补策略 |
|------|-----------------|-------------|---------|---------|
| 数据量 | 112 条 (同质) | 304 条 (异质) | 异质数据更难拟合 | 按家族分层训练 |
| 评估方式 | 90/10 split | Nested CV (5x3) | Nested CV 更严格, R² 通常低 0.03-0.08 | 已是正确做法 |
| 特征质量 | 4 个精选物理特征 | 34 维混合特征 | 物理特征更有信息量 | 补充 Van Krevelen SOL |
| 特征数量 | 4 | 34 | 维度诅咒? | 精选 Top 10-15 特征 |
| 模型 | ExtraTrees | ExtraTrees | 相同 | 调参优化 |

**结论**: 在 Nested CV 下达到 R²=0.97 是**不现实**的。合理目标:
- 304 条 Nested CV: R² = 0.88-0.92
- 加入桥梁+迁移后: R² = 0.90-0.95
- 如果达到 0.93+ 已属优秀

### 6.3 论文创新点总结

基于本调研, 本项目的**三大创新点**:

1. **核酸 Tg 迁移预测** (文献空白): 首次通过合成聚合物 -> 桥梁聚合物 -> 核酸的迁移路径预测核苷酸 Tg
2. **Fox 方程数据增强 + 两阶段训练** (Nature Computational Science 2025 验证了思路): 但我们的实现是独立的、基于树模型的方案
3. **小数据集下的系统消融** (Kaggle 2025 实证树模型 > DL): 在 304 条数据上的完整消融实验, 覆盖特征/模型/数据/迁移四个维度

**额外加分项**:
- Conformal Prediction 为核酸预测提供置信区间 (科学严谨性)
- Stacking 失败的负面结论 (学术诚实, 少见但有价值)
