## 假说 1：构象熵描述符 (Conformational Entropy Descriptor, CED-S)

### 物理依据

**基于 Gibbs-DiMarzio 理论**：Tg 发生在构象熵 S_c → 0 时。

构象熵的定义：
```
S_c = k_B × ln(Ω)

Ω = 聚合物链可访问的构象数
```

当 Ω 减少到一个很小的值时，链段失去了通过构象转变来响应外界扰动的能力 → 玻璃化。

**关键洞察**：如果我们能从单体结构**估算构象多样性**，就直接编码了 Gibbs-DiMarzio 理论的核心参数。

**文献空白**：没有人从 RDKit 的 3D 构象采样来构造 Tg 预测描述符。PerioGT 用了周期性先验但没用 3D 构象信息。MMPolymer 用了 3D 但通过预训练，不是手工特征。

### 具体实现

```python
from rdkit import Chem
from rdkit.Chem import AllChem, rdMolDescriptors
import numpy as np

def compute_conformational_entropy_descriptors(smiles, n_confs=50, n_threads=4):
    """从单体 SMILES 计算构象熵相关描述符"""
    mol = Chem.MolFromSmiles(smiles)
    mol = Chem.AddHs(mol)

    # 1. 生成多个 3D 构象
    params = AllChem.ETKDGv3()
    params.numThreads = n_threads
    params.randomSeed = 42
    cids = AllChem.EmbedMultipleConfs(mol, numConfs=n_confs, params=params)

    if len(cids) < 3:
        return {f'CES_{k}': 0.0 for k in ['diversity', 'energy_spread', 'gyration_var',
                                            'volume_ratio', 'max_dihedral_range']}

    # 2. MMFF 优化
    energies = []
    for cid in cids:
        res = AllChem.MMFFOptimizeMolecule(mol, confId=cid, maxIters=200)
        ff = AllChem.MMFFGetMoleculeForceField(mol, AllChem.MMFFGetMoleculeProperties(mol), confId=cid)
        if ff:
            energies.append(ff.CalcEnergy())

    # 3. 计算描述符
    descriptors = {}

    # CES_diversity: 构象多样性 = 低能构象的 RMSD 分散度
    # 高多样性 → 高构象熵 → 低 Tg
    rmsds = []
    for i in range(min(len(cids), 20)):
        for j in range(i+1, min(len(cids), 20)):
            rmsds.append(AllChem.GetConformerRMS(mol, cids[i], cids[j]))
    descriptors['CES_diversity'] = np.mean(rmsds) if rmsds else 0.0

    # CES_energy_spread: 能量展宽 = 构象能量的标准差
    # 大展宽 → 多样的能量景观 → 构象选择压力大 → Tg 高
    descriptors['CES_energy_spread'] = np.std(energies) if energies else 0.0

    # CES_gyration_var: 回转半径变异性
    # 高变异 → 链构象可塑性高 → 低 Tg
    gyrations = []
    for cid in cids[:20]:
        conf = mol.GetConformer(cid)
        coords = np.array([conf.GetAtomPosition(i) for i in range(mol.GetNumAtoms())])
        centroid = coords.mean(axis=0)
        rg = np.sqrt(np.mean(np.sum((coords - centroid)**2, axis=1)))
        gyrations.append(rg)
    descriptors['CES_gyration_var'] = np.std(gyrations) / np.mean(gyrations) if gyrations else 0.0

    # CES_volume_ratio: van der Waals 体积 / 包络体积
    # 高比值 → 堆积紧密 → 自由体积小 → Tg 高
    vdw_vol = rdMolDescriptors.CalcExactMolWt(mol) / 1.4  # 近似 van der Waals 体积
    # 用最低能构象的包络来估算
    conf = mol.GetConformer(cids[np.argmin(energies[:len(cids)])])
    coords = np.array([conf.GetAtomPosition(i) for i in range(mol.GetNumAtoms())])
    hull_vol = np.prod(coords.max(axis=0) - coords.min(axis=0))  # 包围盒体积
    descriptors['CES_volume_ratio'] = vdw_vol / hull_vol if hull_vol > 0 else 0.0

    # CES_max_dihedral_range: 最大二面角变化范围
    # 关键主链二面角的采样范围 → 直接反映旋转自由度
    # 大范围 → 高柔性 → 低 Tg
    torsion_ranges = []
    torsion_smarts = Chem.MolFromSmarts('[!$([NH]!@C(=O))&!D1]-&!@[!$([NH]!@C(=O))&!D1]')
    matches = mol.GetSubstructMatches(torsion_smarts)
    for match in matches[:5]:  # 最多 5 个关键二面角
        angles = []
        for cid in cids[:20]:
            angle = Chem.rdMolTransforms.GetDihedralDeg(mol.GetConformer(cid), *match[:4]) if len(match) >= 4 else 0
            angles.append(angle)
        if angles:
            torsion_ranges.append(max(angles) - min(angles))
    descriptors['CES_max_dihedral_range'] = max(torsion_ranges) if torsion_ranges else 0.0

    return descriptors
```

**计算复杂度**：
- 每个分子 ~50 构象 × MMFF 优化 ≈ 2-5 秒
- 7,486 个分子 ≈ 4-10 小时（可并行）
- 比 VPD 慢 ~10x，但仍可接受

### 预期效果

- **CES_diversity 预计是强预测因子**：直接编码 Gibbs-DiMarzio 理论核心变量
  - 预期 SHAP 排名: Top 5-10
  - 预期 R² 提升: +0.5-2.0%

- **与现有特征的互补性**：
  - FlexibilityIndex 编码的是"2D 拓扑柔性"（旋转键比例）
  - CES_diversity 编码的是"3D 构象柔性"（实际构象采样的分散度）
  - 两者相关但不等价：一个分子可以有很多旋转键但都被位阻锁住（高 FlexibilityIndex, 低 CES_diversity）

- **预期的共线性**：CES_diversity 与 FlexibilityIndex 相关系数预计 r ≈ 0.5-0.7（相关但不冗余）

### 验证实验

```
实验设计:
  1. 对 7,486 个聚合物计算 5 维 CES 特征 (~5h)
  2. 构建 M2M-V + CES (51d) 特征集
  3. 用 CatBoost + Nested CV 对比:
     - M2M-V 46d (基线): R² = 0.8691
     - M2M-V + CES 51d: 预期 R² = 0.87-0.88
     - Top 15 + CES 20d: 检查 CES 特征能否进入 SHAP Top 15
  4. SHAP 分析: CES_diversity 是否进入 Top 10？

成功标准:
  - R² 提升 ≥ 0.5% (从 0.8691 到 0.874+)
  - CES_diversity 进入 SHAP Top 10
  - CES 特征与 FlexibilityIndex 相关系数 < 0.8 (非冗余)

失败条件:
  - 如果 R² 没有提升 → CES 与现有特征高度冗余
    → 说明 2D 拓扑柔性已足够编码构象熵
    → 需要更精确的构象采样（MD 而非 MMFF）
  - 如果 CES_diversity 和 FlexibilityIndex 相关 >0.9
    → 3D 构象信息不提供增量
    → 转向"旋转势垒估算"（假说 2）
```

## 假说 2：旋转势垒近似描述符 (Rotational Barrier Proxy, RBP)

### 物理依据

**基于自由体积理论和 Schneider-DiMarzio M/f 模型的深化**。

FlexibilityIndex 的局限：它只计数"可旋转键"，但不区分旋转难易程度。例如：
- C-C 单键旋转势垒 ~14 kJ/mol
- C-O 醚键旋转势垒 ~5 kJ/mol
- C-N 酰胺键旋转势垒 ~80 kJ/mol（共振导致部分双键性）
- C-C 与苯环相连时，旋转势垒取决于邻位取代基

**FlexibilityIndex 把所有可旋转键等权计数，但实际上不同键的旋转难度差 10 倍以上**。

如果我们能估算**加权的旋转势垒**，就比 FlexibilityIndex 更精确地编码了链柔性。

### 具体实现

```python
# 基于键类型的旋转势垒查找表 (kJ/mol)
BARRIER_TABLE = {
    'C(sp3)-C(sp3)':      14.0,   # 乙烷型
    'C(sp3)-C(sp2)':      8.0,    # 丙烯型
    'C(sp2)-C(sp2)':      20.0,   # 联苯型
    'C(sp3)-O':           5.0,    # 醚键
    'C(sp3)-N':           12.0,   # 胺键
    'C(sp2)-N(amide)':    80.0,   # 酰胺键 (共振)
    'C-S':                8.0,    # 硫醚键
    'Si-O':               1.0,    # 硅氧烷 (极低)
    'C(sp3)-C(aromatic)': 10.0,   # 苯环连接
    'default':            14.0,   # 默认
}

def compute_rotational_barrier_proxy(smiles):
    """估算加权旋转势垒描述符"""
    mol = Chem.MolFromSmiles(smiles)

    rotatable_bonds = mol.GetSubstructMatches(
        Chem.MolFromSmarts('[!$([NH]!@C(=O))&!D1]-&!@[!$([NH]!@C(=O))&!D1]')
    )

    barriers = []
    for bond_match in rotatable_bonds:
        a1, a2 = mol.GetAtomWithIdx(bond_match[0]), mol.GetAtomWithIdx(bond_match[1])
        # 根据原子类型和杂化查表
        barrier = classify_bond_barrier(a1, a2, BARRIER_TABLE)
        barriers.append(barrier)

    if not barriers:
        return {'RBP_mean': 80.0, 'RBP_max': 80.0,
                'RBP_weighted_flex': 0.0, 'RBP_barrier_diversity': 0.0}

    return {
        'RBP_mean_barrier':     np.mean(barriers),      # 平均旋转势垒
        'RBP_max_barrier':      np.max(barriers),       # 最高势垒 (瓶颈)
        'RBP_weighted_flex':    sum(1/b for b in barriers) / mol.GetNumHeavyAtoms(),
                                # 加权柔性 = Σ(1/barrier) / N_heavy
                                # 物理含义: 每个重原子贡献的旋转自由度，按势垒加权
        'RBP_barrier_diversity': np.std(barriers) / np.mean(barriers) if np.mean(barriers) > 0 else 0,
                                # 势垒多样性 → 反映链是否有"柔-刚交替"结构
    }
```

### 预期效果

- **RBP_weighted_flex 应优于 FlexibilityIndex**：
  - FlexibilityIndex: 所有旋转键权重相同
  - RBP_weighted_flex: 低势垒键权重更大（更物理）
  - 预期 SHAP 替代 FlexibilityIndex 进入 Top 3

- **RBP_max_barrier 编码"瓶颈效应"**：
  - 链段运动取决于最难旋转的那个键（木桶效应）
  - 一个高势垒键就能"锁住"整个链段
  - FlexibilityIndex 完全忽略了这个效应

- 预期 R² 提升: +0.3-1.0%（与现有柔性特征部分冗余但更精确）

### 验证实验

```
实验设计:
  1. 构建 BARRIER_TABLE (从文献/量化计算获取 ~20 种键型的势垒值)
  2. 对 7,486 个聚合物计算 4 维 RBP 特征 (~30 秒，纯 2D 计算)
  3. 对比实验:
     a. M2M-V 46d + RBP 50d
     b. M2M-V (用 RBP_weighted_flex 替换 FlexibilityIndex) 46d
     c. Top 15 + RBP 19d
  4. SHAP 分析: RBP_weighted_flex vs FlexibilityIndex 的排名对比

成功标准:
  - RBP_weighted_flex 替换 FlexibilityIndex 后 R² 不降或提升
  - RBP_max_barrier 进入 SHAP Top 15

失败条件:
  - 如果 RBP_weighted_flex ≈ FlexibilityIndex 效果相同
    → 键类型区分对 Tg 预测不重要
    → 说明"有多少旋转键"比"旋转多难"更重要
  - 备选: 用 RDKit 的 GetTorsionEnergy 做更精确的势垒估算
```

## 假说 3：GC 残差学习 + 物理边界约束 (GC-Residual Learning)

### 物理依据

**基于 Van Krevelen 基团贡献法 + 物理约束**。

两个核心思想：

1. **GC 残差学习**：先用基团贡献法计算 Tg_GC，然后让 ML 模型只学习残差 ΔTg = Tg_exp - Tg_GC。
   - 物理依据：GC 法 R²≈0.99 (Liu et al. 2020)，已经编码了 Tg 的大部分变异
   - ML 只需学习 GC 无法捕捉的**非加和性效应**（链间相互作用、构象效应等）
   - 残差的方差远小于原始 Tg 的方差 → ML 学习任务更简单

2. **物理边界约束**：
   - 100K < Tg < 700K（已知物理范围）
   - Tg < Tm（对半结晶聚合物）
   - 柔性指数增大 → Tg 单调降低（物理规律）

**文献空白**：2025 综述中指出"无 PINN 直接用于 Tg 预测"。GC 残差学习在 Tg 领域未被系统实证。

### 具体实现

```python
def gc_residual_pipeline(X, y_true, gc_tg_values):
    """GC 残差学习 pipeline"""
    # 1. GC 基线预测
    y_gc = gc_tg_values  # 预计算的 GC 法 Tg (Van Krevelen/Liu)

    # 2. 计算残差
    residuals = y_true - y_gc  # 这是 ML 要学习的目标

    # 3. 训练模型学习残差
    model = CatBoostRegressor(
        monotone_constraints={
            'FlexibilityIndex': -1,   # 柔性↑ → 残差↓ (Tg↓)
            'ced_weighted_sum': 1,     # CED↑ → 残差↑ (Tg↑)
        },
        # 物理边界: 残差应在 [-200, +200]K 范围内
    )
    model.fit(X, residuals)

    # 4. 最终预测
    y_pred = y_gc + model.predict(X)
    y_pred = np.clip(y_pred, 100, 700)  # 物理边界硬裁剪

    return y_pred
```

**GC 计算的挑战**：

需要对 7,486 个聚合物进行基团分解。两种方案：
- **方案 a**: 使用 RDKit 的 BRICS 分解或自定义 SMARTS 规则自动识别 58 个 Van Krevelen 基团
- **方案 b**: 使用 Liu et al. 2020 的简化 GC 公式（需要 backbone atom count 和基团查找表）

### 预期效果

- **R² 提升预期: +0.5-2.0%**
  - GC 法本身 R²=0.99 → 残差方差极小
  - ML 在小方差目标上更容易学到信号
  - 但 GC 法对非标准聚合物可能失败（杂环、含硅等）

- **物理合理性提升**：
  - 模型预测将不再给出 Tg < 100K 或 > 700K 的不合理值
  - 单调性约束确保柔性↑→Tg↓ 的物理规律

- **外推能力提升**：
  - GC 法本身有良好的外推性（基团贡献具有可加性）
  - ML 只负责修正，修正量较小 → 外推时不会偏离太远

### 验证实验

```
实验设计:
  1. 对 7,486 个聚合物计算 GC Tg (需要实现基团分解)
  2. 评估 GC 法在我们数据集上的精度 (预期 R² > 0.90)
  3. 对比:
     a. 标准 CatBoost (M2M-V → Tg): 基线
     b. 残差 CatBoost (M2M-V → ΔTg → Tg_GC + ΔTg): 残差学习
     c. 残差 CatBoost + 单调约束: 物理约束版
     d. 残差 TabPFN (M2M-V → ΔTg → Tg_GC + ΔTg): 最强组合
  4. 外推测试: 在核苷酸迁移上对比有无 GC 残差

成功标准:
  - 残差模型 R² ≥ 标准模型 R²
  - 核苷酸迁移误差减小（GC 约束提供了物理锚点）
  - 预测值不超出 100-700K 范围

失败条件:
  - 如果 GC 分解对非标准聚合物覆盖率 < 80%
    → 方案 b: 只对可分解的子集使用残差学习
  - 如果残差模型 R² < 标准模型
    → GC 误差过大导致残差噪声增加
    → 改用 GC 作为额外特征（而非残差目标）
  - 备选: Tg_GC 作为第 47 个特征加入 M2M-V
```

## 与竞争方法的差异化

| 维度 | PolyMetriX | Afsordeh | 本项目(现有) | 本项目(+假说) |
|------|------------|----------|-------------|-------------|
| 柔性编码 | RDKit NumRotatable | FlexibilityIndex | FlexibilityIndex | **RBP 加权势垒** |
| 链段信息 | 主链/侧链拆分 RDKit | 无 | VPD 3-mer | VPD + **CES 构象采样** |
| 物理约束 | 无 | 无 | 无 | **GC 残差 + 单调性** |
| Tg 理论对应 | 无 | 部分 (4 物理量) | 部分 (CED, M/f) | **完整 (自由体积+Gibbs-DM+CED)** |

**论文创新点总结**：
1. VPD (已完成) — 聚合过程信息编码
2. RBP (假说 2) — 从"有多少旋转键"到"旋转有多难"
3. CES (假说 1) — 从 2D 拓扑到 3D 构象空间
4. GC 残差 (假说 3) — 物理先验作为学习基线
5. 核酸迁移 (已完成) — 文献空白的应用创新
