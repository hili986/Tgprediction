# 改进方向分析：从单体描述符到聚合物级表示

> 日期：2026-03-13
> 项目：同济大学 SITP — AI 辅助高分子材料设计（Tg 预测）
> 前置文档：`失败原因分析.md`（确认单体描述符天花板 R²≈0.84）
> 参考调研：`前沿方法调研.md`（40+ 篇前沿文献综述）

---

## 0. 改进方向总览

基于失败原因分析，改进必须突破以下瓶颈中的至少一个：

```
瓶颈 1：表示层面 — 单体 SMILES 无法编码链级/体相效应
瓶颈 2：数据层面 — 304 条 Bicerano 数据不足以支撑复杂模型
瓶颈 3：物理层面 — 描述符缺乏深层物理先验（如高斯链、拓扑约束）
瓶颈 4：目标层面 — 核酸 Tg vs Tm 概念不清晰
```

以下按可行性和潜在收益，分析 6 个改进方向。

---

## 方向 1：图神经网络（GNN）— 自动学习结构特征

### 1.1 核心思路

将聚合物重复单元转换为**分子图**（原子为节点、化学键为边），用 GNN 自动学习结构特征，替代手工描述符。

```
当前方法：SMILES → 手工描述符 (19-34 维) → 树模型 → Tg
GNN 方法：SMILES → 分子图 (原子+键) → GNN 自动提取特征 → Tg
```

### 1.2 文献支撑

| 方法 | 年份 | R² | MAE (K) | 数据量 | 关键创新 |
|------|------|-----|---------|--------|---------|
| polyGNN | 2023 | 0.89 | ~22 | 7,000 | 周期性边界条件 |
| GATBoost | 2024 | 0.91 | ~20 | 6,500 | GAT + XGBoost 混合 |
| GC-GNN | 2024 | 0.90 | ~21 | 5,000 | 高斯链物理先验 |
| Chem-DAGNN | 2022 | 0.95 | RMSE=28°C | PolyInfo | DAG 编码合成路径 |

### 1.3 为什么 GNN 能超越手工描述符

1. **自动特征学习**：GNN 的消息传递机制自动学习原子邻域的结构模式，不需要人工设计 SMARTS 匹配规则
2. **保留拓扑信息**：分子图天然保留原子连接拓扑，而 Morgan 指纹将图哈希为固定维度向量时有信息损失
3. **聚合物特定设计**：polyGNN 的周期性边界条件处理了聚合物的"无限链"特性，比 `*→[H]` 替换更物理合理
4. **可组合性**：GNN 的图嵌入可以与传统描述符拼接（如 GATBoost = GAT + XGBoost），灵活性高

### 1.4 GNN 能否突破单体描述符天花板？

**部分可以**，但有限度：

- GNN 仍然操作在**重复单元分子图**上（不是整条聚合物链）
- 它能学到比手工描述符更丰富的局部结构信息（R²: 0.84 → 0.89-0.91）
- 但链级效应（分子量、立构规整性）和体相效应（结晶度、交联密度）仍然无法从单体图中获得
- **最乐观估计**：单体级 GNN 能达到 R²≈0.91-0.95

### 1.5 对本项目的适用性评估

| 维度 | 评估 |
|------|------|
| **数据需求** | 通常需要 1000+ 样本；304 条可能不够 |
| **计算资源** | 单 GPU 足够训练小模型（~1 小时） |
| **实现复杂度** | 中等（PyTorch Geometric / DGL） |
| **预期提升** | R²: 0.84 → 0.88-0.91 |
| **核酸迁移帮助** | 有限（仍然是单体级表示） |
| **SITP 项目匹配** | ⚠️ 需要额外学习 PyTorch，当前栈是纯 scikit-learn |

### 1.6 推荐子方向

- **GATBoost 路线**（最务实）：用 GAT 提取图嵌入，与现有手工描述符拼接，送入 GBR/XGBoost。复用现有代码，只新增图嵌入模块。
- **polyGNN 路线**（最经典）：直接用开源 polyGNN 代码，在 Bicerano 数据上评估。
- **GC-GNN 路线**（物理引导）：高斯链先验提升外推能力，对核酸预测可能更有帮助。

## 方向 2：Transformer 预训练语言模型

### 2.1 核心思路

将聚合物 SMILES 视为"化学语言"，用 Transformer 在大规模无标注数据上预训练（自监督），学习通用化学语义，再微调到 Tg 预测。

```
预训练阶段：100M+ 聚合物 SMILES → Transformer → 化学语义嵌入 (768 维)
微调阶段：化学语义嵌入 + Tg 标签 (304-5000 条) → 线性回归 → Tg
```

### 2.2 文献支撑

| 方法 | 年份 | 架构 | R² | 预训练数据 | 微调数据 |
|------|------|------|-----|-----------|---------|
| polyBERT | 2023 | DeBERTa | 0.93 | 100M | ~5,000 |
| TransPolymer | 2023 | RoBERTa | 0.88 | — | — |
| MMPolymer | 2025 | Multi-modal | 0.94 | — | — |
| LLM 零样本 | 2024 | GPT/LLaMA | <0.50 | 通用 | 0 |

### 2.3 为什么 Transformer 能超越 GNN

1. **大规模预训练**：polyBERT 在 1 亿聚合物上预训练，学到了丰富的化学模式（官能团共现、结构-性质关联）
2. **迁移学习**：预训练模型即使在 300 条小数据上微调也能保持高精度（知识已在预训练中获得）
3. **端到端**：直接从 SMILES 字符串到性质预测，无需特征工程

### 2.4 Transformer 的局限

- **仍然是单体级表示**：polyBERT 输入的也是重复单元 SMILES，只是用 Transformer 替代了手工描述符
- **黑箱性强**：768 维嵌入不可解释，对论文的"可解释性"部分不利
- **需要 GPU**：预训练需要大量 GPU（但可以用开源预训练权重）
- **已有上限**：polyBERT R²=0.93，MMPolymer R²=0.94，天花板接近

### 2.5 对本项目的适用性评估

| 维度 | 评估 |
|------|------|
| **数据需求** | 微调只需 300+ 条（✅ 满足） |
| **计算资源** | 微调只需单 GPU ~30 分钟（预训练用开源权重） |
| **实现复杂度** | 中高（需要 HuggingFace + PyTorch） |
| **预期提升** | R²: 0.84 → 0.88-0.93 |
| **核酸迁移帮助** | 中等（预训练可能见过核酸相关 SMILES） |
| **SITP 项目匹配** | ⚠️ 技术栈跳跃较大 |

### 2.6 推荐子方向

- **polyBERT 微调**（最直接）：下载开源 polyBERT 权重，在 Bicerano 304 条上微调。代码量小（~100 行），但需要 PyTorch + Transformers 环境。
- **polyBERT 嵌入 + 传统 ML**（最兼容）：用 polyBERT 提取 768 维嵌入，与现有描述符拼接，送入 GBR。保留现有代码框架。
- **TransPolymer 路线**：从头训练更轻量，但精度低于 polyBERT。

## 方向 3：多模态融合（1D SMILES + 3D 结构）

### 3.1 核心思路

结合 1D 文本表示（SMILES）和 3D 分子构型，融合不同模态的信息来预测 Tg。

```
模态 1 (1D)：SMILES → Transformer → 文本嵌入
模态 2 (3D)：3D 构型 → 3D GNN (等变网络) → 3D 嵌入
融合：跨模态注意力 → 联合嵌入 → Tg
```

### 3.2 文献支撑

- **MMPolymer** (AAAI 2025)：R²=0.94，当前 SOTA
  - Star Substitution 策略：将 P-SMILES 的 `*` 转换为可生成 3D 构型的结构
  - 跨模态对齐：1D Transformer + 3D GNN 的特征对齐
- **GeoALBEF** (ACS 2025)：RMSE 降低 8.6%（vs 单模态）
- **Uni-Poly** (npj 2025)：统一多模态多领域表示

### 3.3 为什么多模态有效

1. **3D 信息是互补的**：SMILES 编码的是分子拓扑（连接方式），3D 构型编码的是空间排列（键角、二面角）
2. **构型与 Tg 直接相关**：链的空间构型决定了柔性、自由体积、链间距离 — 这些是 Tg 的直接决定因素
3. **突破 1D 天花板**：纯 1D 方法的上限约 R²≈0.93（polyBERT），加入 3D 可推至 0.94+

### 3.4 多模态的挑战

- **3D 构型生成**：需要通过 RDKit / ETKDG / MD 模拟生成 3D 构型
- **聚合物的 3D 构型问题**：重复单元的 3D 构型不等于聚合物链的 3D 构型（寡聚物近似）
- **计算成本高**：MD 模拟获取 3D 构型可能需要数小时/分子
- **实现复杂度最高**：需要同时实现 1D Transformer + 3D GNN + 跨模态融合

### 3.5 对本项目的适用性评估

| 维度 | 评估 |
|------|------|
| **数据需求** | 需要 3D 构型（可用 RDKit ETKDG 快速生成） |
| **计算资源** | GPU 必需，训练时间数小时 |
| **实现复杂度** | 高（多模块协调） |
| **预期提升** | R²: 0.84 → 0.90-0.94 |
| **核酸迁移帮助** | 中等（3D 构型可能捕获堆积效应） |
| **SITP 项目匹配** | ❌ 技术栈和复杂度超出当前项目范围 |

### 3.6 推荐子方向

- **简化版多模态**：用 RDKit ETKDG 生成 3D 构型，提取 3D 描述符（如 3D-MoRSE、WHIM），与现有描述符拼接送入传统 ML。避免了 GNN/Transformer 的复杂度。
- **MMPolymer 复现**：复现开源 MMPolymer，直接在 Bicerano 数据上评估。工程量大但方法成熟。

## 方向 4：物理引导的机器学习

### 4.1 核心思路

将高分子物理理论作为先验知识嵌入 ML 模型，约束模型的学习空间，提升外推能力和可解释性。

```
纯数据驱动：X → ML → Tg（黑箱）
物理引导：X → ML（约束：Tg 满足 WLF 方程 / 高斯链统计 / 自由体积理论） → Tg
```

### 4.2 可用的物理先验

#### (a) 自由体积理论

```
Tg ≈ f(自由体积分数 fg, 热膨胀系数 α)
fg ≈ 0.025 在 Tg 处（ISO-自由体积假说）
```

- **应用方式**：将计算得到的自由体积估计值作为额外特征
- **现有部分实现**：FractionCSP3 是自由体积的粗略代理

#### (b) 高斯链理论（GC-GNN 使用）

```
链构型统计：<R²> = Cn × n × l²
Cn（特征比）反映链刚性 → 与 Tg 高度相关
```

- **应用方式**：从重复单元结构估算 Cn（特征比），作为额外特征
- **GC-GNN 实证**：将高斯链先验嵌入 GNN 后，外推能力显著提升

#### (c) 基团贡献法作为基线

```
Tg_GC = Σ(基团贡献值)  ← 已知方法，精度 MAE ~30K
残差 = Tg_exp - Tg_GC  ← ML 学习残差
```

- **应用方式**：ML 学习基团贡献法的残差，而非直接学习 Tg
- **优势**：物理基线提供了合理的起点，ML 只需纠正偏差
- **文献**：2023-2024 多篇工作验证了"基团贡献 + ML 残差"优于纯 ML

#### (d) Zhao et al. 精选特征策略

```
4 个物理描述符：FlexibilityIndex, SideChainRatio, Polarity, H-bond
这 4 个特征在 ~700 样本上达到 R²=0.97
```

- **关键洞察**：这些特征编码了比 RDKit 自动描述符更深的物理意义
- **SideChainRatio**：侧链原子/主链原子比，反映链刚性与自由体积的平衡
- **我们缺少的**：SideChainRatio 需要知道哪些原子是主链、哪些是侧链 — 这需要聚合物结构解析

### 4.3 文献支撑

| 方法 | 物理先验 | 效果 |
|------|---------|------|
| GC-GNN | 高斯链理论 | 外推 R² 提升 10-15% |
| 基团贡献+ML | Van Krevelen/Bicerano GC | 优于纯 ML |
| Zhao et al. | 4 个精选物理特征 | **R²=0.97** |
| PINN | WLF 方程约束 | 理论严谨但聚合物应用仍在早期 |

### 4.4 对本项目的适用性评估

| 维度 | 评估 |
|------|------|
| **数据需求** | 无额外需求（✅） |
| **计算资源** | 最低（纯 CPU，scikit-learn 即可） |
| **实现复杂度** | 低-中 |
| **预期提升** | R²: 0.84 → 0.88-0.92（取决于物理先验质量） |
| **核酸迁移帮助** | 低（物理先验主要适用于合成聚合物） |
| **SITP 项目匹配** | ✅ 完全兼容现有技术栈 |

### 4.5 推荐子方向

- **Zhao 特征复现**（最高收益）：实现 SideChainRatio 特征（需要 BigSMILES 主链/侧链解析），结合现有特征达到更高 R²。
- **基团贡献残差学习**（最务实）：用 Van Krevelen 基团贡献法计算 Tg_GC，ML 学习残差。
- **自由体积特征增强**（低成本）：用 RDKit 3D 构型估算分子体积，计算自由体积相关特征。

## 方向 5：迁移学习与数据增强

### 5.1 核心思路

利用大量相关任务或无标注数据增强小数据集的学习效果。

### 5.2 可行策略

#### (a) 自监督 GNN 预训练

```
预训练：50,000+ 无标注聚合物 → GNN + 掩码/对比学习 → 聚合物嵌入
微调：聚合物嵌入 + 304 Tg 标签 → 回归头 → Tg
```

- **Wang et al. (2024)**：小数据集 (N<200) 上 R² 提升 10-15%
- **优势**：利用了 22K 外部无标注数据（我们已有）
- **需要**：PyTorch + GNN 框架

#### (b) 合成数据预训练

```
合成数据：基团贡献法生成 10,000+ 合成 Tg → 预训练 ML
微调：304 真实 Tg → 微调
```

- **Kuenneth et al. (2024)**：小数据集 MAE 降低 15-20%
- **优势**：不需要额外收集数据，用物理模型生成
- **注意**：Phase 2 的 Fox 虚拟数据策略类似但失败了 — 区别在于基团贡献法生成的数据包含更丰富的结构-Tg 关联

#### (c) 多任务学习

```
任务 1：Tg 预测（304 标签）
任务 2：Tm 预测（可用数据更多）
任务 3：密度预测
共享表示 → 各任务头
```

- **polyGNN** 的多任务版本在小数据集上表现更好
- **优势**：Tg/Tm/密度之间存在物理关联，共享表示可以互相增强

#### (d) 外部数据分层迁移（已验证）

我们在 Phase 3 已验证了加权迁移策略：

```
已验证：22K 外部数据 (w=0.1-0.3) + 304 Bicerano → R²=0.898 迁移
```

可以进一步优化：
- 对外部数据做 canonical SMILES 去重
- 引入数据质量评分，按质量加权
- 分域迁移：先识别哪些外部数据与目标最相关

### 5.3 对本项目的适用性评估

| 维度 | 评估 |
|------|------|
| **数据需求** | 已有 22K 外部数据（✅） |
| **计算资源** | 低-中（取决于方法） |
| **实现复杂度** | 低（合成数据预训练）到 中（自监督 GNN） |
| **预期提升** | R²: 0.84 → 0.87-0.90 |
| **核酸迁移帮助** | 中等（更好的通用模型可能外推更好） |
| **SITP 项目匹配** | ✅ 合成数据预训练与现有栈兼容 |

## 方向 6：核酸专用策略

### 6.1 问题重新定义

在考虑核酸 Tg 预测之前，需要明确两个根本问题：

**问题 1：预测目标是什么？**

| 目标 | 物理意义 | 数据可用性 | 可行性 |
|------|---------|-----------|--------|
| 核酸 Tg（玻璃化转变） | 干态 DNA 薄膜的链段运动冻结 | 仅 1-2 个数据点 | ❌ 极低 |
| 核苷酸/核苷 Tg | 小分子晶体的玻璃化转变 | <10 个数据点 | ⚠️ 低 |
| 核酸 Tm（熔解温度） | 双螺旋解链温度 | 数千个数据点 | ✅ 高 |

**问题 2：是否应该放弃核酸 Tg 预测？**

基于失败原因分析，核酸 Tg 预测面临三重困境：
1. 数据极度匮乏（全球 <10 个 Tg 数据点）
2. 物理机制根本不同（碱基堆积 vs 链段运动）
3. 单体描述符完全不适用于聚合物级核酸

**建议**：将核酸预测目标从"Tg 预测"转向以下选择之一。

### 6.2 可选核酸策略

#### (a) 策略转向：核酸 Tm 预测

- 用最近邻模型（Nearest-Neighbor）结合 ML 修正
- 数据量充足（SantaLucia 参数集、MELTING 数据库）
- 但这偏离了 Tg 预测的主题

#### (b) 核苷酸小分子 Tg 预测（已部分成功）

- Phase 3 证明：ATP 1.0K、ADP 1.3K、AMP 9.5K 误差
- 扩展到更多核苷酸类似物：UMP、CMP、各种修饰核苷
- **有限但可行**：小分子 Tg 预测是单体描述符的适用范围

#### (c) DNA-聚合物共轭物 Tg

- DNA-b-PS（DNA-嵌段-聚苯乙烯）等共轭材料
- 聚合物部分的 Tg 可预测，DNA 部分的贡献通过实验确定
- **研究前沿**：文献中缺乏 ML 工作

#### (d) 核酸灵感的氢键聚合物 Tg

- 不预测核酸本身的 Tg，而是预测**受核酸碱基配对启发**设计的合成聚合物的 Tg
- 如：含嘌呤/嘧啶侧基的聚丙烯酸酯
- 桥梁聚合物策略的自然延伸

### 6.3 对本项目的适用性评估

| 维度 | 评估 |
|------|------|
| **数据需求** | 取决于策略（a 高 b 低 c 极低 d 中） |
| **实现复杂度** | 低（b）到 高（a, c） |
| **SITP 项目匹配** | ✅ 策略 b 和 d 最匹配 |
| **论文价值** | 策略 d > c > b > a |

## 关键洞察与建议

### 洞察 1：不一定需要深度学习

Zhao et al. 用 4 个精选物理描述符 + ExtraTrees 达到 R²=0.97，超过了所有 GNN 和 Transformer 方法。这证明**特征工程质量 > 模型复杂度**。在 SITP 本科项目的技术栈约束下，方向 4（物理引导）可能是投入产出比最高的选择。

### 洞察 2：GNN/Transformer 是发表的热点方向

虽然物理引导方法精度可能更高，但 GNN/Transformer 是 2023-2026 年的发表热点。如果目标是发高影响力论文，方向 1/2 + 方向 4 的组合更有吸引力。

### 洞察 3：核酸预测需要重新定义目标

继续用合成聚合物 Tg 模型预测核酸 Tg 是死胡同。需要明确选择新的目标（Tm 预测 / 核苷酸小分子 / 核酸启发聚合物）。

### 洞察 4：数据扩展的杠杆效应

22K 外部数据的迁移学习已将 R² 推至 0.898。进一步优化数据质量（去重、质量评分、分域迁移）可能是低成本高回报的投入。
