## 方案 A：物理特征增强（推荐 ⭐）

### 核心思路

复现 Zhao et al. (2025) 的精选物理描述符策略，用**更有物理意义的特征**替代/补充当前的 RDKit 自动描述符，同时引入基团贡献法残差学习。

### 技术路线

```
步骤 1: 实现 SideChainRatio 特征
  - 从 BigSMILES 解析主链/侧链结构
  - 计算 侧链原子数 / 主链原子数
  - 这是 Zhao et al. 的关键特征之一

步骤 2: 增强物理特征集
  - 保留现有 L1H (34维)
  - 新增: SideChainRatio, 估算 Cn (特征比), 自由体积估计
  - 目标特征集: L1HP (L1H + Physical) ~37-40 维

步骤 3: 基团贡献法残差学习
  - 实现简化 Van Krevelen 基团贡献法计算 Tg_GC
  - ML 目标从 Tg → (Tg - Tg_GC)（学习残差）
  - 物理基线提供合理起点，ML 只需纠正偏差

步骤 4: 结合外部数据优化
  - 22K 外部数据 canonical SMILES 去重
  - 数据质量评分 + 加权训练
  - 目标: 迁移 R² > 0.90

步骤 5: 核酸策略调整
  - 放弃 DNA 聚合物 Tg 预测（数据不足 + 物理不适用）
  - 聚焦核苷酸小分子 Tg（已证明可行：ATP 1.0K 误差）
  - 扩展到 UMP, CMP, 修饰核苷等
```

## 方案 B：GNN 混合路线

### 核心思路

用 GNN（图注意力网络）从分子图中自动提取特征，与现有手工描述符拼接，送入传统 ML 模型。参考 GATBoost (Park et al., 2024, R²=0.91)。

### 技术路线

```
步骤 1: 环境搭建
  - 安装 PyTorch + PyTorch Geometric
  - 实现 SMILES → 分子图转换（原子特征 + 键特征 + 邻接矩阵）

步骤 2: 轻量级 GAT 模型
  - 2-3 层 GAT，隐藏维度 64-128
  - 全局平均池化 → 图嵌入 (64 维)
  - 在 22K 外部数据上预训练（自监督 / 有监督）

步骤 3: 混合特征矩阵
  - GAT 嵌入 (64 维) + L1H 描述符 (34 维) = 98 维
  - 送入 GBR / ExtraTrees

步骤 4: Nested CV 评估
  - 复用现有评估框架
  - 对比: 纯 L1H vs 纯 GAT vs 混合
```

from torch_geometric.nn import GATConv, global_mean_pool

class PolymerGAT(torch.nn.Module):
    def __init__(self, in_channels, hidden_channels=64, num_layers=3):
        super().__init__()
        self.convs = torch.nn.ModuleList()
        for i in range(num_layers):
            self.convs.append(GATConv(
                in_channels if i == 0 else hidden_channels,
                hidden_channels, heads=4, concat=False
            ))

    def forward(self, x, edge_index, batch):
        for conv in self.convs:
            x = conv(x, edge_index).relu()
        return global_mean_pool(x, batch)  # (batch, hidden_channels)

## 方案 C：Transformer 嵌入路线

### 核心思路

利用开源 polyBERT 预训练模型提取聚合物 SMILES 的 768 维化学语义嵌入，与现有描述符拼接或替换，送入传统 ML。

### 技术路线

```
步骤 1: 加载 polyBERT
  - 下载预训练权重 (HuggingFace)
  - 实现 SMILES → polyBERT 嵌入 (768 维)

步骤 2: 嵌入 + 传统 ML（方案 C1）
  - polyBERT 嵌入 (768 维) → PCA 降维 (50-100 维) → GBR
  - 或: polyBERT 嵌入 + L1H (34 维) → GBR

步骤 3: 直接微调（方案 C2，可选）
  - 在 polyBERT 后接回归头
  - 用 304 条 Bicerano 数据微调
  - 需要小心过拟合（早停 + dropout）

步骤 4: Nested CV 评估
```

### 优势

1. **预训练知识** — polyBERT 在 1 亿聚合物上预训练，包含丰富化学语义
2. **实现简单** — 只需加载模型 + 提取嵌入，~50 行代码
3. **精度上限高** — polyBERT 微调可达 R²=0.93
4. **可能对核酸有帮助** — 预训练数据中可能包含核苷酸相关结构

### 劣势

1. **黑箱性最强** — 768 维嵌入完全不可解释
2. **依赖外部模型** — polyBERT 权重可能下载困难（国内网络）
3. **微调过拟合风险** — 304 条数据微调 1 亿参数模型
4. **PyTorch + Transformers 依赖** — 技术栈变化

### 预期时间线

| 步骤 | 估算工作量 |
|------|-----------|
| 环境搭建 + 模型下载 | 1-2 天 |
| 嵌入提取 | 1 天 |
| 传统 ML 集成 | 2-3 天 |
| 微调（可选） | 3-5 天 |
| 评估 + 对比 | 2-3 天 |
| **总计** | **~2-3 周** |

### 预期成果

- 均聚物 CV R²: 0.84 → 0.88-0.93
- 论文角度：预训练语言模型在小数据集 Tg 预测中的迁移效果评估

## 方案 D：数据驱动迁移优化

### 核心思路

在不改变模型架构的前提下，优化数据质量和迁移策略，最大化现有框架的潜力。

### 技术路线

```
步骤 1: 外部数据质量精炼
  - RDKit canonical SMILES 去重（当前是字符串去重，存在遗漏）
  - 异常值检测：移除 Tg 与结构不匹配的数据点
  - 数据源质量评分：PolyMetriX (高) > POINT-2 (中) > NeurIPS (低)
  - 精炼后预计 15,000-18,000 条高质量数据

步骤 2: 分域迁移学习
  - 对外部数据按聚合物类型分类（PA, PU, PI, PE, PS...）
  - 对每个 Bicerano 样本，找到最近的外部数据子集
  - 按相似度加权训练（近域高权重，远域低权重）

步骤 3: 合成数据预训练
  - 用基团贡献法生成 10,000 合成 Tg 数据
  - 预训练 GBR/ET，再用 304 真实数据微调
  - 与 Phase 2 Fox 虚拟数据的区别：基团贡献法引入结构信息

步骤 4: 集成策略优化
  - 放弃 Stacking（已证明在小数据上失败）
  - 改用 Voting (加权平均) 或 Blending (holdout 集元学习)
  - 多模型: GBR + ET + SVR，各在不同数据子集上训练

步骤 5: 核酸扩展
  - 收集更多核苷酸 Tg 数据（文献搜索）
  - 优化桥梁迁移策略
```

### 优势

1. **零技术栈变化** — 100% 兼容现有代码
2. **低风险** — 每步可独立验证
3. **已有部分验证** — 外部迁移 R²=0.898 是良好起点
4. **数据洞察** — 质量精炼过程本身产出有价值的分析

### 劣势

1. **天花板受限** — 仍使用单体描述符，理论上限不变
2. **提升可能有限** — 从 0.84/0.898 到 0.90+ 的边际收益递减
3. **论文新颖性不足** — 数据清洗和加权训练不够"前沿"
4. **不解决根本问题** — 单体描述符天花板仍在

### 预期时间线

| 步骤 | 估算工作量 |
|------|-----------|
| 数据精炼 | 3-4 天 |
| 分域迁移 | 3-4 天 |
| 合成数据预训练 | 2-3 天 |
| 集成优化 | 2-3 天 |
| 核酸扩展 | 1-2 天 |
| **总计** | **~2-3 周** |

### 预期成果

- 均聚物 CV R²: 0.84 → 0.87-0.90
- 迁移 R²: 0.898 → 0.91-0.93
- 论文角度：数据质量与迁移策略对小数据集 Tg 预测的影响

## 方案 E：全栈多模态深度学习

### 核心思路

复现 MMPolymer (AAAI 2025, R²=0.94 SOTA)，实现 1D SMILES Transformer + 3D GNN 的多模态融合。

### 技术路线

```
步骤 1: 环境搭建
  - PyTorch + Transformers + PyG + RDKit 3D

步骤 2: 1D 模块 — SMILES Transformer
  - SMILES tokenizer
  - 预训练 / 加载 polyBERT 权重

步骤 3: 3D 模块 — 等变 GNN
  - RDKit ETKDG 生成 3D 构型
  - SchNet / DimeNet / PaiNN 等变网络

步骤 4: 跨模态融合
  - Star Substitution 策略
  - 跨模态注意力对齐

步骤 5: 端到端训练
  - 多模态联合训练
  - Nested CV 评估
```

### 优势

1. **当前 SOTA** — MMPolymer R²=0.94 是最高精度
2. **3D 信息** — 空间构型可能捕获部分链级效应
3. **论文影响力** — "多模态聚合物表示学习"是前沿热点
4. **开源可复现** — MMPolymer 代码已开源

### 劣势

1. **实现复杂度极高** — 3 个模块 + 跨模态融合
2. **数据量不足** — 304 条数据训练多模态模型高度过拟合风险
3. **计算资源** — GPU 必需，训练时间长
4. **技术栈跳跃太大** — 从纯 scikit-learn 到全套 DL
5. **时间风险** — 调试成本可能超出 SITP 时间预算
6. **可能不如方案 A** — Zhao R²=0.97 > MMPolymer R²=0.94

### 预期时间线

| 环境搭建 | 2-3 天 |
| 1D Transformer | 5-7 天 |
| 3D GNN | 5-7 天 |
| 跨模态融合 | 5-7 天 |
| 训练 + 调参 | 5-7 天 |

- 均聚物 CV R²: 0.84 → 0.90-0.94
