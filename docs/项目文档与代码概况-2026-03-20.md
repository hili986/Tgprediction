#### 问题诊断

| 问题 | 具体表现 | 影响 |
|------|---------|------|
| **内容重叠严重** | "前沿方法调研"、"深度学习方法调研"、"前沿方法深度调研"、"SOTA 深度调研" 四份文件有大量重复 | 新人不知该读哪份 |

| **时效性模糊** | 早期调研（3-07）的结论可能已被后续实验推翻，但文档未标注 | 信息过时误导 |
| **入口缺失** | 没有一份文档告诉读者"先读什么、再读什么" | 新人迷路 |
| **逻辑链隐含** | 失败→改进→方案的链条只有作者知道，文档间无显式链接 | 上下文断裂 |
| **竞赛文档混入** | 统计建模大赛的 2 份文档与主线无关但放在同一目录 | 主题污染 |

## 五、现有系统的优势与不足

### 优势

1. **调研深度扎实**：18 份报告覆盖了 200+ 篇论文，结构化程度高
2. **实验记录完整**：每个实验有 JSON 结果 + 阶段 summary + 总结
3. **代码组织清晰**：分层架构（data→features→ml→gnn），模块边界明确
4. **知识沉淀丰富**：KNOWLEDGE_NOTES.md 积累了 70+ 知识条目
5. **决策可追溯**：失败→改进→方案选择的决策链完整

### 不足

| 类别 | 问题 | 严重度 |
|------|------|--------|
| **导航** | 无总入口、无阅读顺序指引、新人会迷路 | 高 |
| **冗余** | 4 份前沿调研内容大量重叠 | 高 |
| **时效** | 早期文档结论可能已过时，无标注 | 中 |
| **命名** | 文件名风格不统一（日期/无日期/中英混用） | 中 |
| **索引** | 无统一实验编号表、无文档目录 | 中 |
| **状态** | 计划文件的 DRAFT 状态未更新 | 低 |
| **隔离** | 竞赛文档与主线文档混放 | 低 |
| **维护** | 无机制保证文档随代码/实验更新 | 高 |

## 六、整理的核心挑战

1. **如何在保留历史脉络的同时消除冗余？** — 不能简单删除，因为每份调研有其写作时的语境
2. **如何设计"不变的结构"？** — 需要一套目录框架，新内容只在固定位置追加
3. **如何实现"自维持"？** — 需要 hook/配置让文档随代码和实验自动更新
4. **如何服务两类读者？** — 新队友要快速了解全貌，项目负责人要查详细数据
